# 300ボーのモデムが生んだキーバインド——半世紀経っても `hjkl` を押し続ける理由

## はじめに

GitHubの片隅に「呪い」のコードを見つけた。

**vi**。1976年、Bill Joy が C言語で書いたテキストエディタだ。

約10,000行（System V 版）。C 99%。**ADM-3A という端末の上で、300ボーのモデム越しに書かれた。**

始まりは1976年の夏。ロンドン大学の George Coulouris が UC バークレーを訪れ、DECtape に入った `em`（Editor for Mortals——「人間のためのエディタ」）を持ち込んだ。当時の標準エディタ `ed` は、ファイルの中身を画面に表示する機能すら持っていなかった。`em` は画面上でテキストを編集できる、革新的なエディタだった。

バークレーの大学院生 Bill Joy と Chuck Haley はこれに感銘を受け、`em` のコードを元に `en` を作り、さらに拡張して `ex`（extended editor）を生み出した。そして1976年、Joy は `ex` に **ビジュアルモード** を追加した。画面全体を使ってテキストを編集できるモード——これが `vi`（visual）だ。

Joy は自宅の第二次世界大戦の余剰住宅に住んでいた。300ボーのモデムと ADM-3A 端末。画面の描画が追いつかないほど遅い回線。**だからこそ、vi のコマンドは一文字で、先行入力ができるように設計された。** 画面が追いつく前に、次のコマンドを打てるように。

2026年現在、vi（とその子孫 Vim/Neovim）は **POSIX 標準** に含まれ、ほぼ全ての Unix 系システムにインストールされている。50年前に300ボーのモデムのために設計されたキーバインドを、光回線の上で使い続けている。

## 発掘された痕跡

### 痕跡1：ADM-3A の呪い——`hjkl` の由来

ADM-3A は Lear Siegler 社が1976年に製造した端末だ。この端末のキーボードには **専用のカーソルキーが存在しなかった。** 代わりに、`H`、`J`、`K`、`L` のキーキャップに矢印が刻印されていた。

- `H` ← 左
- `J` ↓ 下
- `K` ↑ 上
- `L` → 右

Joy はこの端末で vi を開発した。だから vi は `hjkl` でカーソルを移動する。

さらに、ADM-3A では **Escape キーが現代のキーボードの Tab キーの位置** にあった。ホームポジションから手を動かさずに押せる場所。だから vi はモード切り替えに Escape を使う。

**1976年の端末のキー配置が、2026年のプログラマーのキーバインドを決めている。** これは設計ではなく、呪いだ。

### 痕跡2：300ボーの制約が生んだ哲学

Joy 自身が語っている：

> 「terse な1文字コマンドと、ディスプレイの先を入力できる能力は、ソフトウェアを開発した時の遅い300ボーモデムの結果だ。画面の描画が自分の思考より遅い時に、生産的でいたかった」

300ボー = 毎秒約30文字。80桁 × 24行の画面を全て描画するのに約1分かかる。

だから vi のコマンドは **最小限のキーストロークで最大限の編集を行う** ように設計された。`dd` で行削除、`cw` で単語変更、`3yy` で3行コピー。1文字か2文字。画面が追いつく前に、意図を全て伝えきる。

Joy は MIT の Emacs チームとの対比を語っている：

> 「Emacs を作ってた人たちは MIT の研究室に座って、今で言うファイバーチャネルみたいな接続で PDP-10 につながっていた。無限に速い画面で。一方、俺はバークレーの第二次世界大戦の余剰住宅に座って、モデムと、やっとカーソルが画面の下から動かせる端末を使ってた」

**同じ時代に、同じ問題を解いた二つのエディタ。しかし環境が違った。** MIT の高速回線が Emacs の複雑なキーバインド（Ctrl+Meta+何か）を許し、バークレーの300ボーが vi の極限的な簡潔さを生んだ。

### 痕跡3：コマンドモードの無限ループ

```c
/*
 * This is the main routine for visual.
 * We here decode the count and possible named buffer specification
 * preceding a command and interpret a few of the commands.
 * Commands which involve a target (i.e. an operator) are decoded
 * in the routine operate in ex_voperate.c.
 */

vmain()
{
    register int c, cnt, i;
    ...
    for (;;) {
        TSYNC();
```

`ex_vmain.c`——vi のビジュアルモードのメインループ。**無限ループ** でコマンドを受け取り続ける。各コマンドの前に一時ファイルを同期し（`TSYNC()`）、状態をリセットする。

オペレータ（`d`, `c`, `y` 等）とモーション（`w`, `b`, `$` 等）を **分離して組み合わせる** 設計。`dw`（単語削除）、`c$`（行末まで変更）、`y3j`（3行下までコピー）。**オペレータ × モーション = 操作** というcomposable（組み合わせ可能）な設計は、vi の最大の発明だ。

### 痕跡4：テープドライブが壊れた日

1978年12月、バークレーに VAX が導入された。Joy はマルチウィンドウ機能を vi に追加しようとしていた。

しかし **バックアップが取れなかった。テープドライブが壊れていた。** Joy はバックアップなしで作業を続けた。そしてソースコードが壊れた（"scrunched"）。完全なリスティングも持っていなかった。

> 「ディスプレイコードをウィンドウ用にほぼ全て書き直していた。そこで諦めた」

Joy は以前のバージョンに戻り、ドキュメントを書いて、コードを閉じた。**もしあのテープドライブが壊れていなければ、vi はマルチウィンドウを持っていた。** そしてプログラマビリティも追加するつもりだった。

Vim が vi にマルチウィンドウを追加したのは1996年。**18年遅れた。** テープドライブ1台の故障が、エディタの歴史を18年遅らせた。

### 痕跡5：`ed` → `em` → `en` → `ex` → `vi` の進化

```c
/* Copyright (c) 1981 Regents of the University of California */
static char *sccsid = "@(#)e_c.c	1.1	(9.1	2/9/83)";
```

vi のソースコードには UC Berkeley と AT&T の **二重の著作権表示** がある。`ed` は AT&T の Ken Thompson が書いた行エディタ。George Coulouris がそれを改良して `em` を作った。Joy と Haley が `em` を元に `en`、`ex` を作った。Joy が `ex` にビジュアルモードを追加して `vi` になった。

**5世代のエディタの DNA が、一つのコードベースに混在している。** `ed` のコードを基盤にしたため、vi のソースコードは20年以上 AT&T の商用ライセンスが必要だった。2002年にようやく Caldera が BSD ライセンスで公開した。

### 痕跡6：75人から始まった世界標準

vi は1979年5月の 2BSD（Second Berkeley Software Distribution）に含まれてリリースされた。**最初の配布先は、わずか75人。**

75本の配布テープから始まったエディタが、POSIX 標準に含まれ、50年後のほぼ全ての Unix/Linux/macOS マシンにプリインストールされている。

## エディタ戦争の発端

vi と Emacs の対立は、**1985年以前から** Usenet で始まった。コンピュータ史上最も長く続く「聖戦」だ。

しかし Joy の視点で見ると、これは思想の対立ではない。**環境の違いだ。** MIT の高速回線で生まれた Emacs と、バークレーの300ボーで生まれた vi。同じ問題を、異なる制約の下で解いた。

Joy は vi を作った後、Sun Microsystems を共同設立した。BSD Unix に TCP/IP、仮想メモリ、高速ファイルシステムを実装した男が、vi のことを「あれは副産物だ」と語る。**しかし副産物が、本業より長く生き残っている。**

## 推定される経緯

**1976年夏**: George Coulouris が `em` をバークレーに持ち込む。

**1976年**: Bill Joy と Chuck Haley が `em` を元に `en`、`ex` を開発。Joy が `ex` にビジュアルモードを追加。

**1978年3月**: 1BSD リリース（ex 1.1 を含む）。

**1978年12月**: VAX 導入。テープドライブ故障でマルチウィンドウ版のソースコードを喪失。

**1979年5月**: 2BSD リリース。`vi` の名前で初めてインストール。75人に配布。

**1982年**: Sun Microsystems 共同設立。Joy はエディタ開発を離れる。

**1985年頃**: Usenet 上でエディタ戦争が本格化。

**1991年11月2日**: Bram Moolenaar が Vim（Vi IMproved）をリリース。

**2002年1月**: Caldera が Ancient Unix Code を BSD ライセンスで公開。vi のソースが自由にアクセス可能に。

**2023年8月3日**: Bram Moolenaar 死去（62歳）。Vim の開発はコミュニティに引き継がれる。

## 鑑定人所見

vi は「呪い」だ。

1976年の ADM-3A 端末のキーキャップに刻まれた矢印が、50年後のプログラマーの指を支配している。`hjkl` で移動し、`Esc` でモードを切り替え、`:wq` で保存する。なぜ？ **1976年の端末にカーソルキーがなかったから。** なぜ Escape がモード切り替えなのか？ **1976年の端末で Escape が Tab の位置にあったから。**

これは設計ではない。偶然だ。しかしその偶然が POSIX 標準になり、Vim に引き継がれ、Neovim に引き継がれ、VS Code のVim 拡張機能になり、2026年の IDE でも `hjkl` は生きている。

最も象徴的なのは **テープドライブの故障** だ。1978年12月、Joy はマルチウィンドウを vi に実装しかけていた。テープドライブが壊れ、バックアップが取れず、ソースコードが壊れた。Joy は諦めて旧バージョンに戻った。**もしテープドライブが壊れていなければ、vi は1978年にマルチウィンドウを持ち、エディタの歴史は18年早く進んでいた。**

Plan 9（鑑定書 #046）は「設計図」だった——正しすぎて使われなかった理想の OS。vi は **「呪い」** だ——偶然の産物が半世紀の標準になった。Plan 9 の理想主義と vi の偶然性。どちらが長く生き残ったかは、歴史が答えている。
