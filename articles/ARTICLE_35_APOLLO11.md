# 月面着陸を支えた64KBのコードに「BURN_BABY_BURN」と書いてあった話

## はじめに

GitHubの片隅に「月」のコードを見つけた。

**Apollo 11 AGC**（Apollo Guidance Computer）。1969年、Margaret HamiltonがMIT Instrumentation Laboratoryで率いたチームが書いた、月着陸船の誘導コンピュータプログラムだ。AGCアセンブリ言語。64KBのメモリ（ROM 57KB + RAM 6KB）。2.048MHzのクロック。15ビットのワード長。

このコンピュータが、ニール・アームストロングとバズ・オルドリンを月面に着陸させた。

GitHubで57,000以上のスターを集めているこのリポジトリには、マスター点火ルーチンのファイル名が**BURN_BABY_BURN**、ユーザーインターフェースが**PINBALL_GAME_BUTTONS_AND_LIGHTS**、デバッグコメントに**TEMPORARY, I HOPE HOPE HOPE**と書かれている。

月に行くコードにユーモアがある。

---

## Margaret Hamilton——「ソフトウェアエンジニアリング」を発明した女性

Margaret Hamiltonはコンピュータ科学者であり、数学者であり、NASAの歴史を変えた人物だ。

彼女がMIT Instrumentation Laboratoryでアポロ計画の飛行ソフトウェアを率いたとき、「ソフトウェアエンジニアリング」という言葉はまだ存在しなかった。**Hamiltonが作った言葉**だ。

> *「ハードウェアおよび他の種類のエンジニアリングと区別し、各種エンジニアリングを全体的なシステムエンジニアリングプロセスの一部として扱うため、『ソフトウェアエンジニアリング』という用語を使い始めた。」*

当時、この用語は冗談のネタにされた。ソフトウェアが「エンジニアリング」だなんて。しかし月面着陸の成功後、誰も笑わなくなった。

2016年、バラク・オバマ大統領がHamiltonに**大統領自由勲章**を授与した。

## 発掘された痕跡

### 痕跡1：BURN_BABY_BURN——1960年代ポップカルチャーが月へ

```
BURN_BABY_BURN--MASTER_IGNITION_ROUTINE.agc
```

マスター点火ルーチン——エンジンを点火する最も重要なプログラムの一つ——のファイル名が**BURN_BABY_BURN**。

「Burn, baby! BURN!」は、1965年のロサンゼルスのラジオDJ Magnificent Montagueのキャッチフレーズだ。開発者Don EylesとAdlerが、ロック文化を宇宙船に持ち込んだ。

ファイル内にはラテン語の格言まで埋め込まれている。

**HONI SOIT QUI MAL Y PENSE**（このことに悪意を持つ者には恥あれ）
**NOLI SE TANGERE**（触るな）

月に行くコードにラテン語。

### 痕跡2：PINBALL_GAME_BUTTONS_AND_LIGHTS——ピンボールで月に行く

```
PINBALL_GAME_BUTTONS_AND_LIGHTS.agc
```

これはユーザーインターフェース——キーボード入力とディスプレイ出力を管理するプログラムだ。宇宙飛行士がコンピュータと対話するための**VERB/NOUNコマンドシステム**を実装している。

宇宙飛行士は2桁の数字ペアでコマンドを入力する。

```
Verb 06 Noun 20 = 状態ベクトルを表示
Verb 37 Noun 00 = プログラム変更
```

このインターフェースを「ピンボールのボタンとライト」と名付けたのは、1960年代のアーケードゲーム文化への愛着だ。ミッションクリティカルなシステムに、開発者の個性が息づいている。

### 痕跡3：TEMPORARY, I HOPE HOPE HOPE——開発者の祈り

```
TC BANKCALL # TEMPORARY, I HOPE HOPE HOPE
```

「一時的なもの、だといいな、いいな、いいな」。

デバッグ中の試行錯誤で入れた一時的なコードに、開発者が祈りを込めた。シェイクスピアの『オセロ』のデズデモーナのセリフ「These are portents, but yet I hope, I hope they do not point on me」への文学的参照とも解釈されている。

月面着陸を控えた緊張の中で、開発者は祈っていた。

### 痕跡4：64KBで月に行く——コアロープメモリ

AGCのハードウェア仕様は、2026年の目で見れば信じられない。

| 仕様 | 値 |
|:---|:---|
| RAM | 2,048ワード（約6KB） |
| ROM | 36,864ワード（約57KB） |
| ワード幅 | 15ビット + 1パリティビット |
| クロック | 2.048 MHz |
| メモリサイクル | 11.72マイクロ秒 |

ROMは**コアロープメモリ**——物理的にワイヤーを磁気コアに織り込んで作る。ワイヤーがコアを貫通すれば1、迂回すれば0。文字通り「織り上げられた」プログラムだ。

6KBのRAMと57KBのROM。合計64KB未満。この中に、月着陸船の誘導、ナビゲーション、制御の全プログラムが収まっている。15ビットの固定小数点演算で、月への軌道を計算した。

### 痕跡5：1202アラーム——Hamiltonの設計が月面着陸を救った

1969年7月20日、Eagleが月面に接近する最終段階で、AGCから**1202アラーム**が発生した。

コンピュータオーバーロード——ではない。

Hamiltonが設計した**優先度ベースのタスクスケジューリング**が、正常に動作していた。ランデブーレーダーからの不要なデータ処理で負荷が上がり、AGCは低優先度タスクを切り捨て、着陸に必要な高優先度タスク（ナビゲーション・推進制御）に集中した。

Hamiltonは事前にこう文書化していた——「このアラームはエラーではなく、ステータスレポートだ」。

Mission Controlはアームストロングとオルドリンに指示した。

**「Proceed.」**（継続せよ。）

この設計がなければ、着陸は中止されていた可能性が高い。Hamiltonの「優先度ベースのタスク管理」という設計判断が、月面着陸を救った。

### 痕跡6：OFF TO SEE THE WIZARD——オズの魔法使い

```
TC POSTJUMP # OFF TO SEE THE WIZARD...
CADR BURNBABY
```

「魔法使いに会いに行く……」。着陸レーダーの初期化時のコメント。『オズの魔法使い』への参照だ。

次のコメント：

**SEE IF HE'S LYING**（彼が嘘をついていないか確認せよ）

レーダーデータの二重チェック機能を、ユーモアで説明している。月へ向かうコードの中で、開発者たちはドロシーの冒険を引用していた。

### 痕跡7：15ビット固定小数点——浮動小数点なしで月へ

AGCには浮動小数点演算がない。全ての計算は**15ビットの1の補数固定小数点**で行われる。

ビット14が1/2を表す。プログラマーは暗黙的に小数点位置を追跡する必要がある。乗算するたびにビットがシフトし、精度が変わる。

これで月への軌道計算をやった。

後のIntel 8087（1980年）が浮動小数点演算を一般に普及させるまで、あと11年。AGCの開発者たちは、浮動小数点なしで、月との距離を計算し、着陸速度を制御し、誘導アルゴリズムを実行した。

## 推定される経緯

**1961年**: ジョン・F・ケネディ大統領が「10年以内に月に人間を送る」と宣言。

**1963年**: MIT Instrumentation Laboratoryがアポロ誘導コンピュータの開発を開始。Margaret Hamiltonがソフトウェアチームに参加。

**1966-1968年**: AGCソフトウェアの設計・実装。優先度ベースのタスクスケジューリング、VERB/NOUNインターフェース、誘導アルゴリズムの開発。

**1969年7月16日**: Apollo 11打ち上げ。

**1969年7月20日**: 月着陸。1202アラーム発生。Hamiltonの設計が着陸を救う。アームストロングが月面に第一歩。

**2003年**: VirtualAGCプロジェクトがソースコードのデジタル化を開始。

**2016年7月**: Chris GarryがAGCソースコードをGitHubに公開。即座に57,000スター以上を獲得。

**2016年11月**: Hamiltonが大統領自由勲章を受章。

## 鑑定人所見

Apollo 11 AGCは「人類のコード」だ。

64KBのメモリ、2.048MHzのクロック、15ビットのワード。2026年のスマートフォンの100万分の1以下の性能で、人類は月に行った。そのコードがGitHubで読める。

最も印象的なのは**ユーモア**だ。BURN_BABY_BURN、PINBALL_GAME、OFF TO SEE THE WIZARD、TEMPORARY I HOPE HOPE HOPE——月面着陸という人類史上最も緊張するミッションのコードの中に、開発者たちの笑い声が聞こえる。

そして**1202アラーム**。コンピュータがオーバーロードしたとき、Hamiltonの設計が低優先度タスクを切り捨てて着陸を続行した。「このアラームはエラーではなくステータスレポートだ」という事前の文書化が、Mission Controlの「Proceed」を可能にした。

ソフトウェアエンジニアリングという言葉を作ったのはHamiltonだ。そしてその言葉の意味を——「人命がかかるシステムを、正しく設計し、正しくテストし、正しく文書化すること」を——月面着陸で証明した。

64KBで月に行ける。必要なのは容量ではなく、設計だ。
