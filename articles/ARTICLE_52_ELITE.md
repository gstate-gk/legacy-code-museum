# 22KBに8つの銀河を詰め込んだ二人の学生——そしてソース公開で30年絶縁した

## はじめに

GitHubの片隅に「宇宙」のコードを見つけた。

**Elite**。1984年、ケンブリッジ大学の学生 Ian Bell と David Braben が6502アセンブリで書いた、オープンワールド宇宙シミュレーションゲームだ。

38,244行（注釈付き復元版）。オリジナルは1,244行の密集した BBC BASIC。実行バイナリはわずか **22KB**。

この22KBの中に、**8つの銀河、2,048の惑星系、3Dワイヤーフレームレンダリング、経済システム、戦闘AI** が全て収まっている。

全宇宙が **6バイト** のシードから生成される。

惑星の名前も、経済状態も、政治体制も、人口も——全てが48ビットのシードからアルゴリズムで展開される。保存データもたった6バイト。2,048の惑星系を保存するために、惑星系自体を保存する必要はない。シードだけ覚えていれば、宇宙を再構築できる。

しかしこのゲームの物語は、コードの中だけでは終わらない。1999年、Bell が一方的にソースコードを公開した。**Braben は激怒し、二人は約15年間絶縁した。**

---

## 6バイトの宇宙

Elite の全宇宙は、3つの16ビットシード `(s0, s1, s2)` = **6バイト** から生成される。

```asm
.TT54
 LDA QQ15     \ X = s0_lo + s1_lo
 CLC
 ADC QQ15+2
 TAX

 LDA QQ15+1   \ Y = s0_hi + s1_hi + carry
 ADC QQ15+3
 TAY

 LDA QQ15+2   \ s0 = s1
 STA QQ15
 LDA QQ15+3
 STA QQ15+1
 LDA QQ15+5   \ s1 = s2
 STA QQ15+3
 LDA QQ15+4
 STA QQ15+2

 CLC          \ s2 = s0 + s1 + s2 (Tribonacci)
 TXA
 ADC QQ15+2
 STA QQ15+4
 TYA
 ADC QQ15+3
 STA QQ15+5
 RTS
```

**Tribonacci 数列**（3項フィボナッチ）。`s0' = s1, s1' = s2, s2' = s0+s1+s2`。1惑星あたり4回ツイスト。256惑星で1銀河。銀河ジャンプは各バイトを左ローテート。8回転で元に戻る。

**6バイトから2,048の惑星系が全て決定論的に生成される。**

## 発掘された痕跡

### 痕跡1：32個の二文字ペアで全惑星に名前をつける

惑星名は32通りの二文字ペア（ダイグラム）を3〜4個連結して生成される——

```asm
.QQ16
 EQUS "AL"   \ Token 128
 EQUS "LE"   \ Token 129
 EQUS "XE"   \ Token 130
 EQUS "GE"   \ Token 131
 EQUS "ZA"   \ Token 132
 EQUS "CE"   \ Token 133
 EQUS "BI"   \ Token 134
 EQUS "SO"   \ Token 135
 ...
 EQUS "LA"   \ Token 149
 EQUS "VE"   \ Token 150
```

`LA`(149) + `VE`(150) = **LAVE**（ゲーム開始惑星）。

シードの `s2_hi` の下位5ビットがトークン番号、`s0_lo` のbit6が名前の長さ（3ペアか4ペアか）を決定する。**2,048の惑星名が全てこの32ペアの組み合わせから生成される。** 追加データは1バイトも必要ない。

### 痕跡2：3Dワイヤーフレーム——2MHzでバックフェイスカリング

BBC Micro の CPU は **2MHz の 6502**。この上で3Dワイヤーフレームを描画する。

描画ルーチン `LL9` は12段階——

1. 視野チェック（z座標の符号、距離閾値192）
2. オリエンテーションベクトル取得
3. **バックフェイスカリング（ドット積）** — 見えない面を計算せずスキップ
4. 頂点の3D→2D投影
5. 可視エッジの抽出
6. **XOR描画** — 前フレームの線を消去して再描画（フリッカーレス）

バックフェイスカリングにより、ワイヤーフレームが「透けない」。2MHz の CPU でリアルタイム3D——1992年の Wolfenstein 3D（鑑定書 #033）の **8年前**。

### 痕跡3：出版社に「複雑すぎる」と拒否された

Bell と Braben は最初に Thorn EMI に持ち込んだ。

返答——**「複雑すぎる。ゲームは3つのライフで10分で終わるべきだ。」**

当時のゲームは「スコアを競う」アーケードスタイルが主流だった。「終わりのない宇宙を自由に探索する」という概念自体が理解されなかった。

Acornsoft の社長 David Johnson-Davies が引き受けた。前払い **£1,000** + 7.5%のロイヤリティ。

結果——全プラットフォーム合計で **約100万本** を販売。

### 痕跡4：変数名は2〜4文字の暗号

オリジナルのソースコードは、徹底的に短縮された変数名で埋め尽くされている——

`QQ15`, `INWK`, `TT54`, `TT20`, `LL9`, `Ghy`

22KBに全てを収めるため、変数名も関数名も極限まで短くした。40年後、Mark Moxon が COVID-19 ロックダウン中に38,244行の注釈付き復元版を作成するまで、このコードは「完全に解読不能」だった。

### 痕跡5：ソース公開で共同創業者が絶縁した

1999年11月、ゲーム15周年に Bell が自身のウェブサイトで **ソースコードを一方的に公開した**。

**Braben は激怒した。**

Braben は異議申し立てを行い、一時期ソース公開は取り下げられた。二人は **約15年間絶縁** した。

Bell の主張——「ゲームを解放したかっただけだ。最低限『Elite elements copyright Bell and Braben, used with permission』と書いてくれればいい」。

2012年、Braben が Kickstarter で「Elite: Dangerous」を発表する頃、二人はようやく和解した。「以前のような協力関係ではないが、口をきくようになった」。

### 痕跡6：No Man's Sky の直接の祖先

Sean Murray（No Man's Sky 開発者）自身が語っている——

> *「Elite、Star Control、Freelancerが手本。広大さと自由を再現したかった。」*

6バイトのシードから2,048の惑星系を生成した1984年のアルゴリズムが、2016年の No Man's Sky の18京の惑星を生成するアルゴリズムの直接の祖先だ。

### 痕跡7：Braben は Raspberry Pi を共同設立した

David Braben は2012年、**Raspberry Pi Foundation** の共同設立者の一人として、低価格教育用コンピュータを立ち上げた。

1984年に22KBの BBC Micro でプログラミングを学んだ男が、2012年に「子供たちが安くプログラミングを学べるコンピュータ」を作った。Elite → Frontier → Raspberry Pi → Elite: Dangerous。

2014年、Braben は **OBE（大英帝国勲章）** を受章した。

**1982年秋**: Bell と Braben、ケンブリッジ大学 Jesus College で出会う。

**1983年6月**: Braben が3Dワイヤーフレーム描画ルーチンを完成。

**1984年9月**: Elite リリース。BBC Micro版。£1,000前払い + 7.5%ロイヤリティ。

**1984年〜**: 17プラットフォームに移植。約100万本販売。

**1993年**: Braben が Frontier: Elite II をリリース。

**1999年11月**: Bell がソースコードを一方的に公開。Braben 激怒。約15年間絶縁。

**2012年**: Braben が Raspberry Pi Foundation 共同設立。

**2012年11月**: Elite: Dangerous Kickstarter（£1,578,316調達）。Bell と和解。

**2014年**: Braben、OBE受章。

**2020年〜**: Mark Moxon が COVID ロックダウン中に38,244行の注釈付き復元版を作成。

## 鑑定人所見

Elite は「星図」だ。

6バイトのシードから、8つの銀河と2,048の惑星系が展開される。惑星名は32通りの二文字ペアの組み合わせ。経済、政治、人口——全てがアルゴリズムで決まる。22KBの中に、無限の宇宙が畳み込まれている。

最も象徴的なのは **Tribonacci 数列** だ。`s2' = s0 + s1 + s2`。この単純な漸化式が、2,048の惑星系を一意に決定する。データを保存する必要がない——シードを覚えていれば、宇宙を再構築できる。これは1984年の最も美しいアルゴリズムの一つだ。

しかし最も心に残るのは **二人の学生の物語** だ。ケンブリッジで出会い、22KBに宇宙を詰め込み、「複雑すぎる」と拒否され、100万本を売り、そしてソース公開をめぐって絶縁した。Bell はコードを「解放」したかった。Braben は自分たちの共同著作物を守りたかった。どちらの気持ちも理解できる。

Braben は後に Raspberry Pi を共同設立した。1984年に22KBでプログラミングを学んだ男が、2012年に「子供たちが安くプログラミングを学べるコンピュータ」を作った。Elite の精神——制約の中で最大の創造をする——が、次の世代に受け継がれた。

WorldWideWeb（鑑定書 #039）は「源流」だった。Gosling Emacs（鑑定書 #040）は「火種」だった。Elite は **「星図」** だ——6バイトから展開される無限の宇宙。そしてその星図をめぐって、二人の創造者は30年間、別々の軌道を描いた。
