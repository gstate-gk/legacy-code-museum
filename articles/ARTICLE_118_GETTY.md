## はじめに

```c
#define ERASE  '#'
#define KILL   '@'
```

`#` が一文字消去、`@` が行全体消去だ——Backspace でも Ctrl+U でもない。タイプライターで文字を打ち直す時代の慣習がそのままコードに残っている。Bell-32V の `getty.c` は238行。`init.c`（#105）が fork し、認証を `login`（#102）に渡す前の番人だ。

## `ERASE = '#'` と `KILL = '@'`——タイプライター時代の行編集

```c
#define ERASE  '#'
#define KILL   '@'

} else if (c==ERASE) {
    if (np > name)
        np--;
    continue;
} else if (c==KILL) {
    putchr('\r');
    putchr('\n');
    np = name;
    continue;
}
```

Backspace キー（ASCII 8）も Delete キー（ASCII 127）も使わない——`#` で直前の1文字を消し、`@` で行全体を消す。これは Unix 以前のタイプライター端末の慣習だ。物理的に文字を消せないタイプライターでは、`#` を打つことで「前の文字は無効」と伝えた。`@@#@` と入力すれば「行全体を消してさらに1文字消した」という記号的な表現になる。

`stty erase ^H` や `stty kill ^U` で変更できるようになったのはずっと後のことだ。1979年のデフォルトがここに刻まれている。

## ボーレート巡回テーブル——Enter で次の速度へ

```c
struct tab {
    char  tname;    /* このテーブルの名前 */
    char  nname;    /* 次のテーブルへの名前 */
    int   iflags;   /* 初期フラグ */
    int   fflags;   /* 最終フラグ */
    int   ispeed;   /* 入力速度 */
    int   ospeed;   /* 出力速度 */
    char *message;  /* ログインメッセージ */
} itab[] = {

    '0', 1, ..., B300,  B300,  "\n\r\033;\007login: ",
    1,   2, ..., B1200, B1200, "\n\r\033;login: ",
    2,   3, ..., B150,  B150,  "\n\r\033:\006\006\017login: ",
    3,  '0',..., B110,  B110,  "\n\rlogin: ",
    ...
};
```

ダイアルアップ端末のボーレートは事前にわからない。`getty` は `B300` から始め、`getname()` がゼロを返す（ユーザーが何も入力せずEnterを押した）たびに `tabp->nname` で次のエントリへ進む。300→1200→150→110→300 と巡回する。

ユーザーは Enter を繰り返して文字化けが解消されるボーレートを探す——1979年の自動認識だ。`tname` と `nname` を連結した単方向リストでテーブルを構成する設計が、後の `gettytab` ファイル形式に発展した。

## `partab[128]`——7ビット端末へのパリティ付加

```c
char partab[] = {
    0001,0201,0201,0001,0201,0001,0001,0201,
    0202,0004,0003,0205,0005,0206,0201,0001,
    /* ... 128バイト ... */
    0000,0200,0200,0000,0200,0000,0000,0201
};

putchr(cc)
{
    char c;
    c = cc;
    c |= partab[c&0177] & 0200;  /* bit7をパリティに応じてセット */
    write(1, &c, 1);
}
```

ASCIIは7ビット——8ビット目（0200=128）はパリティビットとして使われた。`partab[]` は各文字のビット数が奇数か偶数かを事前計算した128バイトのテーブルで、偶数パリティが必要な文字にはエントリの上位ビット（0200）が立っている。`putchr()` はそのビットを文字にORして送信する。

`0202` は `'\n'`（改行）のエントリ——ビット1が立ち「2つのストップビット」を意味する（`NL` 遅延フラグと組み合わせ）。`0004` は `'\t'`（タブ）、`0003` は `'\r'`（CR）——端末の機械的な動作に合わせた遅延制御だ。

## `/* table 'l' -- LSI Chess Terminal */`——チェス端末専用エントリ

```c
/* table 'l' -- LSI Chess Terminal */
    'l', 'l',
    ANYP+RAW/*+HUPCL*/, ANYP+ECHO/*+HUPCL*/,
    B300, B300,
    "*",
};
```

`itab[]` の最後のエントリは "LSI Chess Terminal" 専用だ。ログインメッセージは `"*"` の1文字——チェス端末のプロンプトがアスタリスクだったからだ。`HUPCL`（Hang Up on CLose）フラグはコメントアウトされている。

Bell Labs には実際にチェスプログラムを動かす専用端末があった——`getty.c` のボーレートテーブルにその端末が刻まれている。`/* table 'i' -- Interdata Console */` も同様に、特定のハードウェアへの対応がコードに直接書かれている。

## `'\033;'` と `'\033:'`——端末固有のエスケープ列

```c
"\n\r\033;\007login: "   /* 300baud: ESC; BEL login: */
"\n\r\033;login: "       /* 1200baud: ESC; login: */
"\n\r\033:\006\006\017login: "  /* 150baud: ESC: ACK ACK SI login: */
"\n\rlogin: "            /* 110baud: 端末依存なし */
```

`\033` は ESC 文字、`\007` は BEL（ビープ音）、`\006` は ACK、`\017` は SI（Shift In）。Datascope 端末に対して `ESC ;` はカーソルを先頭行へ移動し、`ESC :` は消去コマンドだ。各ボーレートに対応する端末の種類が異なり、それぞれに最適な初期化シーケンスが埋め込まれている。

110 ボーの最低速エントリは端末固有の制御なしの素の `"\n\rlogin: "` だ——低速の端末には凝ったシーケンスを送らない割り切り。

## `c == ' '` → `c = '_'`——スペースをアンダースコアに変換

```c
} else if(c == ' ')
    c = '_';
*np++ = c;
```

Unix のユーザー名にスペースは使えない。`getty` の `getname()` ではスペース入力をアンダースコアに変換してから格納する——パスワードファイルのコロン区切りフォーマットを壊さないための防衛だ。今日でも Unix のユーザー名にスペースが使えない慣習はここから続いている。
