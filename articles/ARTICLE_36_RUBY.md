# 鳥取県から世界へ——まつもとゆきひろが「プログラマの幸福」のために書いた言語の話

## はじめに

GitHubの片隅に「幸福」のコードを見つけた。

**Ruby**。1995年、まつもとゆきひろ（Matz）が日本で設計・実装したプログラミング言語だ。C言語で書かれたインタプリタ。「全てがオブジェクト」。ブロックとイテレータ。オープンクラス。

しかしRubyの本質は技術仕様にはない。Matzがこの言語に込めた**設計思想**にある。

> *「I hope to see Ruby help every programmer in the world to be productive, and to enjoy programming, and to be happy. That is the primary purpose of the Ruby language.」*
> 「世界中のプログラマーが生産的になり、プログラミングを楽しみ、幸福になることを願う。それがRuby言語の第一の目的だ。」

プログラマの**幸福**を設計思想の中心に据えた言語。コンピュータのためではなく、人間のために書かれた言語。

---

## Matz——Perlの次の誕生石

まつもとゆきひろは1965年、大阪府に生まれ、4歳から鳥取県米子市で育った。

プログラミング言語に魅せられた少年は、やがて自分の言語を作りたいと思うようになった。影響を受けた言語は多い——Perl、Smalltalk、Lisp、Eiffel、Ada。しかしどれも「完璧」ではなかった。

1993年2月24日、Matzは同僚の石塚圭樹とオンラインチャットで新しい言語の名前を決めた。

**Perl**は6月の誕生石（真珠、Pearl）。Matzが作る言語は、Perlの「次」——7月の誕生石、**Ruby**（ルビー）。

名前は遊び心だが、意図は真剣だった。Perlを「より良く、より美しく、よりオブジェクト指向的に」改善した言語を作る。

1995年12月21日、Matzは**fj.sources**（日本のUsenetニュースグループ）でRuby 0.95をリリースした。鳥取県から、世界へ。

## 発掘された痕跡

### 痕跡1：「プログラマの幸福」——技術仕様ではなく、哲学

Rubyの公式サイトには、技術的な特徴よりも先に**哲学**が語られている。

> *「Ruby is designed to make programmers happy.」*

Matzはインタビューでこう述べている。

> *「フラストレーションの最小化。努力の最小化。そして何より、プログラミングを楽しむこと。」*

これは1995年の言語設計としては異例だった。C++は「効率」を、Javaは「安全性」を、Perlは「実用性」を第一に掲げていた。**「幸福」**を掲げた言語は、Rubyが初めてだ。

### 痕跡2：「驚き最小の原則」——ただしMatzの驚き

Rubyの設計原則として有名な「驚き最小の原則（Principle of Least Surprise）」には、重要な但し書きがある。

> *「The principle of least surprise ... is the principle of least Matz's surprise.」*
> 「驚き最小の原則は……Matzの驚きが最小になる原則だ。」

全ての人にとっての「驚き最小」ではなく、**Matz個人にとって「納得できる」設計**を優先する。この正直さが、Rubyの一貫性を支えている。

Matzが「直感的」と感じる設計は、Smalltalk、Lisp、Perlの経験から培われた直感だ。つまりRubyの設計は、一人の人間の**審美眼**に基づいている。

### 痕跡3：「全てがオブジェクト」——Smalltalkの正統な後継者

Rubyでは、文字通り全てがオブジェクトだ。

```ruby
42.class          #=> Integer
true.class        #=> TrueClass
nil.class         #=> NilClass
"hello".reverse   #=> "olleh"
3.times { puts "Ruby!" }
```

整数、真偽値、nil（null）——他の言語で「プリミティブ」として扱われるものも、全てオブジェクト。Smalltalk-80（鑑定書 #021）の思想を最も忠実に受け継いだ言語の一つだ。

Matz自身が述べている——Rubyは「Smalltalkのようなメッセージパッシングシステムを採用し、すべてのメソッド呼び出しはメッセージ送信として機能する」。

### 痕跡4：ブロックとイテレータ——Lispの魂をRubyの体に

Ruby 0.95（1995年の最初のリリース）の段階で、**ブロックとイテレータ**は既に実装されていた。

```ruby
[1, 2, 3].each do |n|
  puts n * 2
end

File.open("data.txt") do |f|
  f.each_line { |line| puts line }
end
```

**yield**キーワードでブロックに値を渡す。コレクション側が「どのように反復するか」を知っていて、クライアントコードは「何をするか」だけを記述する。

これはLispの高階関数からの影響だ。しかしLispの括弧地獄ではなく、Rubyの**do...end**で包む。読みやすく、書きやすく、幸福。

### 痕跡5：複数言語からの戦略的取捨選択

Matzは複数の言語から**意図的に**影響を取り入れた。

| 言語 | Rubyに取り入れたもの |
|:---|:---|
| Perl | 実用性、正規表現、テキスト処理 |
| Smalltalk | オブジェクトモデル、メッセージパッシング |
| Lisp | ブロック、クロージャ、ミックスイン（Flavorsから） |
| Eiffel | begin/endキーワード、条件分岐構文 |
| Ada | 信頼性への考慮 |

しかしMatzは「直交性の危険性」——任意の機能を何でも組み合わせられることの混乱——を認識していた。複数の良い機能から**1つを選択する戦略**を採用した。足し算ではなく、引き算の設計。

### 痕跡6：fj.sources——日本のUsenetから世界へ

1995年12月21日、Matzは**fj.sources**でRuby 0.95をリリースした。

fj.sources——日本のUsenetニュースグループ。当時、日本のプログラマーがソフトウェアを配布する主要な手段だった。インターネットはまだ黎明期。GitHubは存在しない。SourceForgeすらない。

翌日からruby-listメーリングリストが開始された。

Ruby 0.95の段階で既に実装されていた機能：
- オブジェクト指向（クラス、継承）
- ミックスイン
- イテレータ（ブロック付きメソッド）
- クロージャ
- 例外処理
- ガベージコレクション

同時代のPython 1.x（1991年初版）と比較しても、極めて高度な機能セットだ。Matzが複数の成熟した言語から精選した設計を、短期間で統合した結果。

### 痕跡7：Ruby on Rails——「幸福」が世界を変えた日

2004年、デンマーク人のDavid Heinemeier Hansson（DHH）がRuby on Railsをリリースした。

Railsの成功の鍵は、Rubyの言語機能にあった。

- **Convention over Configuration** — Rubyの「驚き最小」哲学の実践
- **Active Record** — Rubyのオープンクラスと動的メソッド定義の活用
- **DSL（ドメイン特化言語）** — Rubyのブロック構文が自然言語のような記述を可能にした

Twitter（初期版）、Airbnb、GitHub、Shopify——世界的なWebサービスがRailsで構築された。

鳥取県で一人のプログラマーが「幸福のために」設計した言語が、デンマーク人の手でWebフレームワークになり、世界のインターネットインフラを支えることになった。Matzが1993年に思い描いた「Perlの次」は、想像を遥かに超える場所に辿り着いた。

## 推定される経緯

**1965年**: まつもとゆきひろ、大阪府生まれ。4歳から鳥取県米子市で育つ。

**1993年2月24日**: 同僚の石塚圭樹とオンラインチャットで「Ruby」と命名。

**1993-1995年**: 約2年間の開発。C言語でインタプリタを実装。

**1995年12月21日**: fj.sourcesでRuby 0.95をリリース。ruby-listメーリングリスト開始。

**2000年**: 英語での書籍『Programming Ruby』（PickAxe本）出版。国際的な認知が始まる。

**2003年**: チューリング賞受賞者のAlan Kayが「OOPの本質はメッセージパッシング」と語る。Rubyはその思想を最も忠実に実装した言語の一つ。

**2004年**: David Heinemeier HanssonがRuby on Railsをリリース。Web開発の革命。

**2005-2010年**: RailsバブルによりRubyの人気が爆発。Twitter、GitHub、Airbnbが採用。

**2012年**: Ruby 2.0リリース。国際標準（ISO/IEC 30170:2012）化。

**2020年**: Ruby 3.0リリース。「Ruby 3x3」（Ruby 2の3倍速）を達成。

## 鑑定人所見

Rubyは「幸福のコード」だ。

SQLiteには「祝福」があった。Rubyには「幸福」がある。どちらも技術仕様ではなく、**人間への願い**がコードの根幹にある。

最も印象的なのは、Matzの**正直さ**だ。「驚き最小の原則は、Matzの驚きが最小になる原則だ」。普遍的な原理ではなく、一人の人間の審美眼に基づいていると明言する。この正直さが、Rubyの一貫性を30年間支えてきた。

CP/Mは「技術で勝ってビジネスで負けた」。Smalltalkは「発明して売れなかった」。SQLiteは「売らないことで全てに入り込んだ」。Rubyは第四の道を選んだ——**「幸福」を掲げて、共感で広がった**。

Perlの次の誕生石。鳥取県のfj.sourcesから世界へ。まつもとゆきひろが1993年に思い描いた「プログラマが幸せになる言語」は、2026年の今も幸せを生み続けている。
