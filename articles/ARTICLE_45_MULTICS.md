# 失敗した夢が全てのOSを生んだ——妻が帰省中の夏に、去勢されたMulticsが産声を上げた

## はじめに

GitHubの片隅に「夢の廃墟」を見つけた。

**Multics（MULTiplexed Information and Computing Service）**。1965年、MIT と Bell Labs と GE の三者が共同で開発を始めたオペレーティングシステムだ。

10,943ファイル。PL/I 6,816ファイル。ALM アセンブリ785ファイル。35年間稼働し、2000年10月30日に最後のマシンがシャットダウンされた。

リング型プロテクション。階層ファイルシステム。仮想メモリ。動的リンク。ACL（アクセス制御リスト）。リレーショナルデータベース。**現代のOSの概念のほぼ全てが、ここにある。**

しかし Multics は「失敗した夢」と呼ばれた。

1969年、Bell Labs が撤退した。残された Ken Thompson は、妻が息子を連れて1ヶ月帰省した夏に、廃棄寸前の PDP-7 に向かって **3週間でカーネル、シェル、エディタを書いた**。

その名は UNICS——**UNiplexed Information and Computing Service**。Multics の対義語。発音は "eunuchs"（宦官）——**「ボールのない Multics」**。

Brian Kernighan のジョークだった。

---

## コンピュータを電気のように

Fernando Corbató——愛称「Corby」——は夢を持っていた。

**「コンピュータ公益事業」**。電気や水道のように、コンピューティングを誰でも使えるサービスにする。1963年、MIT の Project MAC でその夢に着手した。

GE がハードウェア（GE-645）を、Bell Labs が約25名のエンジニアを送り込んだ。最高の頭脳が集結した。

しかし Multics は巨大になりすぎた。Sam Morgan の言葉——

> *「Multics は同時に多くの木に登ろうとしている。」*

Doug McIlroy の評——

> *「ベトナムのように、彼（副社長）は勝利を宣言して撤退した。」*

## 発掘された痕跡

### 痕跡1：リング型プロテクション——あなたのCPUの中のMultics

あなたの PC の CPU には「カーネルモード」と「ユーザーモード」がある。x86 では Ring 0 と Ring 3 と呼ばれる。

**この概念は1964年の Multics 設計から来ている。**

Multics は8つのリング（Ring 0〜7）を設計した。GE-645 ではソフトウェアエミュレーション。後継の Honeywell 6180 でハードウェア実装された。

ソースコードの ringbr_.pl1——各セグメントが「何番のリングからアクセスできるか」を管理する。リングブラケットは `fixed bin(3)` ——0〜7の3ビット値。

現代の x86 CPU は Ring 0〜3 を持つが、実際には Ring 0（カーネル）と Ring 3（ユーザー）しか使われない。Multics の8リングは過剰だった。Unix は2リングに「引き算」した。

### 痕跡2：階層ファイルシステム——`>` から `/` へ

Multics のファイルパスは `>` で区切られた。

```
>system_library_1>source>initializer.pl1
```

Unix はこれを `/` に変えた。

```
/system_library_1/source/initializer.pl1
```

Thompson の証言——

> *「私が Multics から実際に持ってきたのは、階層ファイルシステムとシェル——置き換え可能な独立プロセスとしてのシェル。」*

`ls`, `cp`, `mail`, `login`——これらのコマンド名は Multics コマンドの直系の子孫だ。

### 痕跡3：「These segs are no more」——詩的なコメント

初期化コード initializer.pl1 に——

```pl1
    call real_initializer;
    call delete_segs$delete_segs_init;
    slt.first_init_seg = 32768;     /* These segs are no more. */
    slt.last_init_seg = 32768;
    call init_proc;                 /* And call out to ring 1, never to return */
```

「これらのセグメントはもう存在しない。」そして「Ring 1 に呼び出して、二度と戻らない。」

Ring 0（最特権）から Ring 1 への制御移譲の瞬間が、たった2行のコメントで語られている。

### 痕跡4：ACL——50年前のパーミッション

acl.pl1 のコメント履歴は **1973年** から始まる。

現代の全ての OS のパーミッション——Linux の chmod、Windows の ACL、macOS のファイル保護——の元祖が、このコードの中にある。

ITS（鑑定書 #032）は「パスワードがない世界」だった。Multics は真逆だ——**全てを保護する世界**。同じ MIT で、同じ時代に、正反対のセキュリティ思想が並立していた。

### 痕跡5：MRDS——MySQLの20年前のリレーショナルDB

1976年6月、Multics は **MRDS（Multics Relational Data Store）** をリリースした。世界最初の商用リレーショナルデータベースの一つだ。

MySQL が誕生するのは1995年。**20年先を行っていた。**

### 痕跡6：PL/I——「まだ存在しない言語」で書いたOS

Multics の開発言語に PL/I が選ばれた。しかし Doug McIlroy の証言——

> *「PL/I はまだ存在しなかったが、説得力ある完全な仕様書があった。」*

存在しない言語でOSを書くために、Bell Labs の McIlroy と Morris が **EPL（Early PL/I）** コンパイラを自作した（1966-67年）。

言語を作ってからOSを書く——Multics のスケールの大きさを象徴するエピソードだ。

### 痕跡7：百万ドルが屋根裏に放置された

Bell Labs 撤退直後の状況——

> *「百万ドル相当の機器が屋根裏に放置され、3人の男に遊ばれていた。」*

その3人の中に、Ken Thompson がいた。

## 妻が帰省中の夏

1969年の夏。

Bell Labs は Multics プロジェクトから撤退した。Thompson は自分の Space Travel（宇宙旅行）ゲームを GE-645 から PDP-7 にポーティングしたかった。しかしそのためにはファイルシステムが必要だった。

妻の Bonnie が生後間もない息子を連れて、カリフォルニアの実家に **1ヶ月間** 帰省した。

Thompson は「一時的な独身生活」を活かして、PDP-7 に向かった。

**カーネル、シェル、エディタを、それぞれ1週間ずつ、計3週間で書いた。**

同じ夏、1969年7月20日、人類は月面に降り立った。そして Bell Labs の屋根裏では、一人の男が全てのOSの祖先を書いていた。

Dennis Ritchie のコメント——

> *「我々はエラー回復コードを全て除外した。エラーがあれば panic() を呼び出し、マシンがクラッシュし、ホールに向かって『再起動しろ』と叫ぶ。」*

Multics は全てを保護しようとした。Unix は全てを捨てた。**引き算が革命を生んだ。**

---

## 「失敗するシステムの作り方について」

Fernando Corbató は1990年にチューリング賞を受賞した。

受賞講演のタイトルは——**"On Building Systems That Will Fail"（失敗するシステムの作り方について）**。

> *「複雑さには難問を複合させる性質がある。シンプルさとエレガンスの価値を重視せよ。」*

> *「野心的なシステムを始めるには楽観主義者でなければならないため、完成時間の過小評価が常態となる。」*

Multics への反省が込められていた。しかし皮肉なことに、Multics は「失敗」していない。**35年間稼働し、2000年10月30日17時08分（UTC）にカナダ国防省で最後のマシンがシャットダウンされた。**

「失敗した夢」は、35年間動き続けた。

2019年7月12日、Corbató 死去。享年93歳。

## 鑑定人所見

Multics は「廃墟の神殿」だ。

10,943ファイル。35年間。現代のOSの概念のほぼ全てがこの中にある。リング型プロテクション、階層ファイルシステム、仮想メモリ、動的リンク、ACL、RDBMS——しかしこの「全て」を詰め込んだがゆえに、Multics は「失敗した」と呼ばれた。

最も象徴的なのは **引き算の美学** だ。Multics は8リングのプロテクションを持っていた。Unix は2つに減らした。Multics はエラー回復コードを書いた。Unix は「panic() を呼んでクラッシュさせろ」と言った。Multics は PL/I という言語を自作してまでOSを書いた。Unix は C という小さな言語で書き直した。

Thompson が妻の帰省中の3週間でカーネル・シェル・エディタを書いた話は、ソフトウェア開発の永遠の教訓だ。Multics は数百人が何年もかけた。Unix は一人が3週間で書いた。しかし Thompson は Multics を経験していなければ、Unix を書けなかった。**「失敗した夢」を経験した人間だけが、その夢を正しく「引き算」できた。**

Corbató のチューリング賞講演「失敗するシステムの作り方について」は、Multics への反省であり、同時に、全ての大規模システム開発者への遺言だ。「シンプルさとエレガンスの価値を重視せよ」——しかし Multics 自身は、その教訓を体現できなかった。

UNICS という名前は Kernighan のジョークだった。「去勢された Multics」。しかしこの「去勢」こそが革命だった。全てを持っていた Multics から、必要なものだけを残して、残りを捨てた。引き算が革命を生んだ。

Wolfenstein 3D（鑑定書 #033）は「扉」だった。Multics は **「廃墟の神殿」** だ——崩れ落ちた夢の中に、現代の全てのOSの設計図が眠っている。
