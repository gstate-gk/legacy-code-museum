# 「dBase I」は存在しない——dBase II という壮大な嘘と.DBFの不死

## はじめに

世界中のGISソフトウェアが、今日も1978年の深夜に書かれたデータ構造を読み込んでいる。

dBase II。1981年に登場したこのPC用データベースには、最初から奇妙な秘密が隠されていた。「II」という数字だ。dBase I は存在しない。最初から「II」として売り出された——それが戦略だった。

Wayne Ratliff は後にこう振り返っている。「dBASEという名前は昔考えたことがあったが、"debasing（格下げ）"に聞こえるので却下していた。」マーケティング担当の Hal Pawluk が「バージョン1は信頼されない」と判断し、最初から「dBase II」という名前をつけた。小文字の「d」と大文字の「BASE」の組み合わせは Pawluk の発案。Ratliff は「天才的だ」と褒めた。

それだけではない。会社名「Ashton-Tate」の「Ashton」は架空の名前だ。George Tate と Hal Lashlee（= Lashlee-Tate では語呂が悪い）がスタートアップを始めるにあたって、「Ashton」を捏造した。

嘘から始まった会社が、嘘のバージョン番号を持つソフトウェアで、PC データベース市場を70%制した。そして1978年の深夜に書かれたファイルフォーマットは、今も死なない。

## 発掘された痕跡

dBase II のソースコードは、Ashton-Tate（後にBorland）によって一貫してプロプライエタリに管理されており、公開されていない。Internet Archive に v2.4 のバイナリ（CP/M版）が保存されているが、8080アセンブリのソースは現存が確認されていない。

しかしコードの代わりに、より豊かな「痕跡」が残っている。

### 痕跡1：火星から居間のPCへ——知の連鎖

Wayne Ratliff は1976年、NASA バイキング計画のフライトチームメンバーとして火星探査機の着陸を支援した。火星のデータを管理するために、Ratliff は MFILE というシステムをすでに書いていた。

その Ratliff が 1978年、社内フットボール賭けに勝つためだけに「Vulcan」を書いた。

起源はさらに遡る。Fred Thompson と Jack Hatfield（JPL）が1971年に書いた **JPLDIS**（Jet Propulsion Laboratory Display Information System）がある。Jeb Long がそれを FORTRAN で UNIVAC 1108 上に移植した。Ratliff はその JPLDIS のアイデアを参考に、8080アセンブリで Vulcan を実装した。

```
JPLDIS（1971, JPL, パブリックドメイン）
    ↓
Vulcan（1978, Wayne Ratliff, 8080 ASM）
    ↓
dBase II（1981, Ashton-Tate, $695）
    ↓
dBase III / IV / Clipper / FoxPro（1980年代）
```

この連鎖が、1990年に決定的な意味を持つ。Ashton-Tate が Fox Software を著作権侵害で訴えた裁判で、判事は **JPLDIS がパブリックドメインであることを根拠に Ashton-Tate の著作権を無効と判決**した。「盗んだ」のではなく「巨人の肩の上に立った」——しかしその巨人はパブリックドメインだった。

### 痕跡2：1978年1月29日深夜——.DBFの誕生

Ratliff が CREATE コマンドの実装を完成させた夜、最初の `.DBF` ファイルが生まれた。

この設計は単純にして革命的だった。**ファイル自身がデータ構造を持つ**。ヘッダーにフィールド定義が書かれており、プログラムは事前知識なしにデータの構造を読み取れる。SQL もスキーマ管理ツールも必要ない。ファイルを開けば、そこに全てが書いてある。

```
.DBF ファイル構造（1978年以来不変の基本設計）:
[ヘッダー: バージョン番号、レコード数、フィールド数]
[フィールド記述子配列: フィールド名、型、長さ]
[データレコード: 削除フラグ + フィールドデータ]
```

この構造が1985年に「デファクトスタンダード」と呼ばれ、その後 **ESRI の Shapefile**（GIS の世界標準フォーマット）の属性テーブルとして採用され、LibreOffice Calc でも読み書きできる。2025年現在も、世界中のGISソフトウェアが1978年夜に生まれたヘッダー構造を解釈している。

### 痕跡3：$50 → $695——価格の逆説

Ratliff は最初、Vulcan を $50 で売っていた。売れなかった。

George Tate が「$695 に上げろ」と言った。実際にその価格で爆発的に売れた。

BYTE誌1980年7月のレビュー（Jerry Pournelle）は「**infuriatingly excellent（腹立たしいほど優秀）**」と書いた。「パワフルだが、ドキュメントが plain lousy（ひどい）。テーブルを叩きながら怒鳴り続けた」。この罵倒混じりのレビューが売上を押し上げた。

1982年9月、IBM PC 版（MS-DOS）が出荷された。IBM 自身が dBase II の移植を依頼していた。これは IBM PC で使えた最初の主要プロフェッショナルアプリケーションのひとつだった。

### 痕跡4：「dBase I」という存在しないバージョン

マーケティングの天才的詐術、もう少し詳しく見る。

1981年当時、Pawluk の論理は明確だった。バージョン1は「最初の試み」に見える。バージョン2は「改善された製品」に見える。買い手は「枯れたソフト」を好む。だから最初から「II」にする。

会社名「Ashton-Tate」も同じ発想だ。二人の人名からなる会社名は「歴史ある組織」に見える。「Ashton」は存在しない人物だが、看板に載っていれば信頼を生む。

これらは嘘だが、嘘をつかないソフトウェアよりも遥かに多くのユーザーを助けた。

### 痕跡5：「ローディングドック男」——創業者を軽視した瞬間から帝国は崩れる

1984年、Ashton-Tate の新 CEO Ed Esber が Wayne Ratliff に言い放った。

**「あなたのアシュトン・テートへの貢献は、倉庫でトラックに箱を積む男と同じだ。」**

創業者・発明者への侮辱。その後の歴史は皮肉だ。

Esber の指揮下で開発された **dBase IV** は 450,000行・75人の開発者・2年の遅延という惨劇になった。市場シェアは63%から40%以下に落ち、1989年には $60M 超の損失。Esber は1990年5月1日に辞任した。

Borland は1991年に Ashton-Tate を **$440M** で買収した。

Ratliff はその前年（1990年）に Ashton-Tate を離れ、FoxPro チームと組んで新しい仕事を始めていた。皮肉にも、Fox Software は Ashton-Tate が著作権侵害で訴えた会社だった。

### 痕跡6：アセンブリからCへ——dBase IIIの秘密

dBase III は C で書き直された。ただし**自動変換ツール**で。

生成されたコードは「**undocumented and syntactically inhuman（ドキュメント不能かつ人間的に異常な構文）**」と関係者に形容された。速度はアセンブリ版より低下したが、移植性は飛躍的に向上した。これが 1980年代後半の複数プラットフォーム展開を可能にした。

Ratliff の手で書かれた人間的なアセンブリコードは、機械によって「inhuman」なC言語に変換されて消えた。

## 年表

**1971年**: JPL（NASA）の Fred Thompson・Jack Hatfield が JPLDIS 開発（パブリックドメイン）。

**1976年**: Wayne Ratliff、バイキング計画で火星データ管理システム MFILE を書く。

**1978年**: Ratliff、社内フットボール賭けのために JPLDIS を参考に「Vulcan」を 8080 ASM で開発。.DBF フォーマット誕生。

**1980年**: BYTE誌7月号に Jerry Pournelle の「腹立たしいほど優秀」レビュー掲載。George Tate が Ratliff に接触。

**1981年**: Ashton-Tate 設立。Vulcan を「dBase II」として $695 で発売。「dBase I」は存在しない。

**1982年9月**: IBM PC（MS-DOS）版出荷。IBM 自身が依頼した移植。PC データベース市場を制覇。

**1984年**: dBase III 発売（C言語・自動変換コード）。Esber が Ratliff に「ローディングドック男」発言。

**1986年**: dBase IV 開発開始（75人体制）。

**1988年**: dBase IV 出荷——バグだらけ、2年遅延。

**1989年**: 市場シェア急落、$60M 超損失。Fox Software との著作権訴訟敗訴（JPLDISはパブリックドメイン判決）。

**1990年5月**: Esber 辞任。Ratliff 退社。

**1991年**: Borland が Ashton-Tate を $440M で買収。dBase 帝国の終焉。

**現在**: .DBF フォーマットは ESRI Shapefile・LibreOffice Calc 等で生存中。

dBase IV の「inhuman C言語」変換コードは、技術的負債の教科書だ。8080アセンブリで人間が書いたコードを、機械が C に変換し、人間が読めなくなった。さらに dBASE 独自のコマンド体系（.USE、.LIST、.FIND 等）はSQLとは別の「方言」として、Clipper・FoxPro 等に引き継がれた。

## 鑑定人所見

dBase II の本質はコードではなく、**設計の哲学**にある。

「ファイルが自分自身の構造を持つ」——この.DBFの設計は、SQLデータベースが普及した現代でも消えていない。なぜか。GIS（地理情報システム）の世界では、ファイル単体で完結するデータフォーマットが今も有用だからだ。ESRI Shapefile は.DBFをそのまま使っている。Google Earth のデータも、気象データも、測量データも、1978年夜のヘッダー構造を今日も読み込む。

Wayne Ratliff はフットボール賭けに勝てたのかどうか、記録は残っていない。しかし彼が書いた .DBF ファイルは、確実に「勝ち続けている」。

**帝国は $440M の赤字で終わり、創業者は「荷積み男」と呼ばれ、著作権は無効とされた。それでも .DBF フォーマットだけは死ななかった。ソフトウェアの遺産とは、コードではなくデータ構造に宿る——dBase II がそれを証明した。**
