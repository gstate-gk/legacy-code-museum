# 14歳でテルミットを調合した少年が、22歳でFPSを発明した——37,361行の固定小数点革命

## はじめに

GitHubの片隅に「革命」のコードを見つけた。

**Wolfenstein 3D**。1992年、id Software の John Carmack が C 言語と8086アセンブリで書いた、世界初の商業的に成功した FPS（First Person Shooter）だ。

37,361行。C言語88%、アセンブリ12%。**浮動小数点演算は一切使わない。** 全ての3D計算を16.16固定小数点で行う。SIN/COS はルックアップテーブル。壁は全て直角。マップは64×64タイルの2Dグリッド。

この制約の中で、Carmack はプレイヤーが3D空間を歩き回る世界を作った。レイキャスティング——画面の各ピクセル列に光線を1本ずつ飛ばし、壁にぶつかった距離の逆数で高さを決める。実際は2Dの計算だが、画面には3Dの城が広がる。

そしてこのコードを書いた男は、14歳の時に **テルミットとワセリンの混合物** で学校の窓ガラスを溶かして侵入し、コンピュータを盗もうとして逮捕された。精神科の評価書には「他人への共感がない、**脚の生えた脳**」と記されていた。

## テルミットから3Dエンジンへ

John Carmack。1970年カンザス州生まれ。

14歳。仲間と学校に忍び込み、Apple II を盗もうとした。テルミット（酸化鉄とアルミニウムの粉末）をワセリンに混ぜた粘着物質を窓ガラスに塗りつけ、溶かして穴を開けようとした。太った共犯者が穴を抜けられず窓を開けてしまい、サイレントアラームが作動。逮捕。1年間の少年院送致。

釈放後、両親が Apple II を買い与えた。

ミズーリ大学に2学期だけ在籍し、中退。ルイジアナ州シュリーブポートの Softdisk 社に就職。ここで **John Romero**、Tom Hall、Adrian Carmack（血縁関係なし）と出会った。

1990年9月のある深夜、Carmack は PC でのスムーズスクロール技術を完成させた。Tom Hall と朝5:30まで作業し、**Super Mario Bros. 3 の第1面を PC で再現するデモ** を作った。タイトルは「Dangerous Dave in Copyright Infringement（著作権侵害のデンジャラスデイブ）」。

Nintendo に持ち込んだが、拒否された。代わりに Commander Keen を作った。これが id Software 誕生のきっかけとなった。

1991年2月1日、id Software 設立。

## 発掘された痕跡

### 痕跡1：浮動小数点ゼロの3D世界

Wolfenstein 3D のコードに浮動小数点演算は **1行もない**。

```c
#define GLOBAL1      (1l<<16)    // 1.0 = 65536
#define TILEGLOBAL   GLOBAL1     // 1タイル = 65536固定小数点単位
#define TILESHIFT    16l         // 右シフト16で整数タイル座標
```

16.16固定小数点。32ビット整数の上位16ビットが整数部、下位16ビットが小数部。80387コプロセッサなしの 386/486 で高速に動作する。

固定小数点乗算は、C の中にインラインアセンブリが直書きされる——

```c
fixed FixedByFrac (fixed a, fixed b)
{
asm  mov  si,[WORD PTR b+2]   // 符号を保存
asm  mov  ax,[WORD PTR a]
asm  mov  cx,[WORD PTR a+2]
asm  or   cx,cx
asm  jns  aok:                 // 負数なら
asm  neg  cx
asm  neg  ax
asm  sbb  cx,0
asm  xor  si,0x8000            // 符号を反転
```

2の補数処理を手動で書く。`#pragma warn -rvl` で「return値なし警告」を黙らせる。**C言語の中にアセンブリが侵食している。**

### 痕跡2：SIN/COS を一度も計算しない

```c
fixed  far sintable[ANGLES+ANGLES/4],
       far *costable = sintable+(ANGLES/4);
```

`ANGLES = 360`。SIN テーブルを360個用意し、COS テーブルは同じ配列の90度ずれたポインタ。三角関数を一度も計算せず、配列参照だけで済ませる。

Spacewar!（鑑定書 #027）の5命令 LFSR と同じ哲学——**計算しないことが最速だ。**

### 痕跡3：コンパイルドスケーラー——実行時に機械語を生成する

Carmack が「今では Bad Thing」と認めた技術。壁の高さ（1〜256ピクセル）ごとに、専用のスケーリング機械語コードを **リアルタイムで生成** してメモリに書き込む——

```c
unsigned BuildCompScale (int height, memptr *finalspot)
{
    // mov al,[si+src]  ← x86機械語を直接書き込む
    *code++ = 0x8a;
    *code++ = 0x44;
    *code++ = src;

    // mov [es:di+heightofs],al
    *code++ = 0x26;
    *code++ = 0x88;
    *code++ = 0x85;
    *((unsigned far *)code) = startpix*SCREENBWIDE;
```

C言語のコードが、x86の機械語バイトを配列に書き込んでいく。書き込まれた機械語は、そのまま関数として呼び出される。

**1992年のJITコンパイル。** キャッシュのない386では最速だった。Carmack の README——

> *「動的にコンパイルされたスケーリングルーティンは今では悪手。キャッシュのない486以前のマシンでは最速だったが、現代プロセッサではコードキャッシュを破壊する。」*

### 痕跡4：セルフモディファイングコード——自分自身を書き換えるレイキャスター

レイキャスティングのアセンブリコア `AsmRefresh`（739行）に、さらに過激な最適化がある——

```asm
    mov   [BYTE cs:horizop],OP_JGE   ; patch a jge in
    mov   [BYTE cs:vertop],OP_JLE    ; patch a jle in
```

**実行中のコード自体を書き換える。** ジャンプ命令のオペコードを動的にパッチして、レイの方向（0-89°, 90-179°, 180-269°, 270-359°）に応じた分岐を実現する。

if 文の代わりに、コード自体を書き換えて分岐コストをゼロにする。現代のセキュリティでは許されない技術が、1992年のゲームでは「正解」だった。

### 痕跡5：VGA のハードウェアを直接叩く

```c
asm  cli                        // 割り込み禁止
asm  mov  cx,[displayofs]
asm  mov  dx,3d4h               // CRTC address register
asm  mov  al,0ch
asm  out  dx,al                  // ハードウェアに直接出力
```

OS を介さず、VGA の CRTC レジスタを直接操作してページフリップする。割り込みを禁止（`cli`）してからハードウェアに書き込む。**カーネルの仕事を、ゲームが直接やっている。**

### 痕跡6：ベトナム戦争元捕虜からの手紙

リポジトリに `GOODSTUF.TXT` というファイルがある。ファンからの手紙が収録されている。

1992年8月9日、ベトナム戦争の元捕虜からの手紙——

「捕虜収容所からの脱走経験があるため、1ヶ月プレイを躊躇した。ゲームをプレイするたびに悪夢が減ってきた。」

そして Microsoft のエンジニアから——

「廊下を歩いていると、"Mein Leben"（ゲームオーバー音声）が聞こえてこないオフィスがない。」

### 痕跡7：シェアウェア——フロッピー1枚の革命

Wolfenstein 3D の第1エピソード（全6エピソード中）は **無料** だった。5.25インチフロッピー1枚に収まるよう設計された——文字通り、友達にコピーを渡せる。

1992年末までにシェアウェアコピーが推定100万枚配布。登録販売は15万枚。

「盗んでもいい。むしろ配ってくれ。面白ければ金を払う。」——GW-BASIC の Gates が「盗むな」と訴えた16年後、id Software は **盗ませることをビジネスモデルにした**。

## Carmack と Romero の決裂

Carmack は技術の天才だった。Romero はデザインの天才だった。

二人は完璧なコンビだった——Wolfenstein 3D、DOOM、Quake。しかし1996年、Romero は id Software を去った。自らの会社 Ion Storm で「Daikatana」を開発。広告には「John Romero's about to make you his bitch」と書かれた。Daikatana は商業的に大失敗した。

Carmack は id Software に残り、DOOM 3、Rage を作り、2013年に退社して Oculus VR（後の Meta）に入った。

2022年12月、Carmack は Meta を去った。退社メモ——

> *「我々は莫大な人員とリソースを持ちながら、常に自己妨害と労力の浪費をしている。組織は私が満足できる効率の半分で動いている。」*

14歳でテルミットを調合した少年は、52歳で Meta に「組織が腐っている」と書いて去った。一貫したキャラクターだ。

## 鑑定人所見

Wolfenstein 3D は「扉」だ。

37,361行のコードが、3Dゲームという新しい世界への扉を開けた。それまでのゲームは上から見下ろすか、横から見ていた。Wolfenstein 3D は **中に入れた**。プレイヤーは城の廊下を歩き、角を曲がり、ナチスの兵士と撃ち合った。

最も印象的なのは **浮動小数点演算が1行もない** ことだ。16.16固定小数点、ルックアップテーブル、インラインアセンブリ——3Dの幻想を、整数演算だけで作り上げた。SIN/COS を一度も計算しない。計算しないことが最速だという哲学は、Spacewar!（鑑定書 #027）の5命令 LFSR に通じる。

コンパイルドスケーラーは、1992年の JIT コンパイルだ。Cのコードが x86 の機械語バイトを配列に書き込み、それを関数として呼び出す。セルフモディファイングコードはジャンプ命令のオペコードを動的にパッチする。現代のセキュリティでは許されない技術が、当時は「正解」だった。Carmack 自身が「今では悪手」と認めている——この知的誠実さは、Ritchie の "astonishing peculiarity"（鑑定書 #028）に通じる。

`GOODSTUF.TXT` のベトナム元捕虜の手紙は、ゲームの力を示している。「ゲームをプレイするたびに悪夢が減ってきた」——コードは人を救うことがある。

ITS（鑑定書 #032）は「聖地」だった——「情報は自由であるべきだ」の聖地。Wolfenstein 3D は **「扉」** だ——22歳の天才が固定小数点で開けた、3D世界への扉。そしてその扉の向こうには、DOOM と Quake と、現代の全ての FPS が待っていた。
