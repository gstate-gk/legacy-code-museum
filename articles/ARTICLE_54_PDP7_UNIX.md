# 妻が息子を連れて帰省した3週間で、夫は全てのOSの祖先を書いた

## はじめに

GitHubの片隅に「起源」のコードを見つけた。

**PDP-7 Unix**。1969年、Ken Thompson が PDP-7 アセンブリで書いたオペレーティングシステムだ。

約3,000行。64ファイル。アセンブリ 82%。**メモリに載るプロセスは、常に1つだけ。**

1969年3月、AT&T は MIT・GE との共同プロジェクト Multics から撤退した。Thompson と Dennis Ritchie にとって、ドアが閉まった瞬間だった。巨大で野心的なOS——セグメント化メモリ、リング型プロテクション、動的リンク——その全てが手の届かない場所に行った。

Thompson は廊下の隅で埃をかぶっていた PDP-7 を見つけた。同僚が回路解析のために買い、使い終わって放置されていたマシンだ。8K ワード（約18KB）のメモリ。彼はまず、そこに宇宙ゲーム **Space Travel** を移植した。

そして1969年夏、妻の **Bonnie** が生まれたばかりの息子を連れて、Thompson の実家があるカリフォルニアに帰省した。3週間の独身生活。Thompson は1週間ずつ、**ファイルシステム、シェル、エディタ**を書いた。**アセンブラ**もこの時期に作った。

これが Unix だ。Multics の夢が潰えた男が、妻の帰省中に、捨てられたマシンの上で書いた OS。この3,000行から、Linux、macOS、Android、そしてインターネットを動かすほぼ全てのサーバーが生まれた。

## 発掘された痕跡

### 痕跡1：「ready」——世界最初のシェルプロンプト

```
shell:
   lac d1
   sys write; ready; 1
shell1:
   lac delim
   sad newln
   jms rline
   jms getcom
```

`sh.s`——世界最初の Unix シェル。プロンプトは **`ready`** だった。`$` でも `%` でもない。「準備できました」と丁寧に告げるシェル。

このシェルは既に `<` と `>` によるリダイレクションと、`&` によるバックグラウンド実行をサポートしていた。1969年のコードに、2026年の bash でも使われるシンタックスが存在する。

```
   sad amper
   jmp shell1
```

`&`（amper）を検出したら、子プロセスの終了を待たずに次のコマンドを受け付ける。**バックグラウンド実行の原型が、ここにある。**

### 痕跡2：fork は27行だった

```
. = orig+020
   1f
   iof
   dac u.ac          " save user AC
   lac 020           " save user return addr
```

カーネル `s1.s` のシステムコール処理。`sys fork` の実装は **わずか27行の PDP-7 アセンブリ** だった。

現在のプロセスをディスクのスワップ領域にコピーし、子プロセスをメモリに残す。それだけ。しかし fork の導入以前、シェルはコマンドを実行するたびに **自分自身を上書きして消滅** していた。コマンドが終わると、ブートストラップがシェルを再読み込みする。fork は、シェルが「生き残る」ことを可能にした最初のシステムコールだ。

Dennis Ritchie が fork-exec モデルの基本概念を提案した。Thompson が27行で実装した。**この27行が、全ての Unix 系 OS のプロセスモデルの起源だ。**

### 痕跡3：メモリに1プロセスだけ

```
" Only one process is in memory at any time
" (in the high half of the 8KW memory)
```

カーネルノート（`NOTES.md`）にある制約。8K ワードのメモリのうち、上半分がユーザープロセス用。**常に1つのプロセスしかメモリ上に存在できない。** マルチプロセスは、ディスクへのスワップアウトで実現する。

```
   lac uquant            " load user quantum (tick) count
   jms betwen; d0; maxquant  " check if between 0 & maxquant (30 jiffies)
      jms swap           "  no: quantum expired: swap process out
```

プロセス量子（タイムスライス）は **30 jiffy**（60Hz クロックで0.5秒）。タイムスライスを使い切ると、プロセス全体をディスクに追い出し、次のプロセスをディスクから読み込む。

10プロセスを管理する OS が、メモリには1プロセスしか置けない。Multics（鑑定書 #034）がリング型プロテクションと仮想メモリで「正しく」解こうとした問題を、Thompson は **力技のスワップで解いた**。

### 痕跡4：紙からスキャンして復元されたコード

```
" ** 11-56-91.pdf page 30
" sh
```

全てのソースファイルの冒頭に、**PDF のページ番号** が記されている。このコードは Bell Labs のプリンタで印刷されたリスティングを、紙からスキャンして復元したものだ。

経緯はこうだ。Dennis Ritchie の死後（2011年）、遺族から寄贈された論文の中に、**「Unix Book II」** と手書きされた黒いバインダーが見つかった。約190ページの PDP-7 アセンブリコードのリスティング。Computer History Museum の David C. Brock が発見した。

Norman Wilson が1980年代後半に Bell Labs でフォトコピーした「Unix Book I」も存在する。2019年10月には、欠落していたソースを含む **第二のノートブック** が新たに発見された。

Warren Toomey（Unix Heritage Society）がこれらのスキャンから OCR とコミュニティの手作業で、実行可能なコードに復元した。**2020年1月、Living Computer Museum のシリアルナンバー113の PDP-7A 実機で、このコードが起動した。**

### 痕跡5：`login:` と `password:` の誕生

```
   lac d1
   sys write; 1f; 4
   ...
1: <lo>; <gi>; <n:>; < 012>
```

`init.s`——世界最初の Unix ログインシーケンス。`login:` と `password:` のプロンプトを表示し、パスワードファイルと照合する。

特殊文字の処理も既にある。`@` が行消去、`#` がバックスペース。認証が成功すると、ユーザーのホームディレクトリに `chdir` し、シェルをメモリアドレス 4096 にロードして実行する。

**UID -1 がスーパーユーザー。** 後に UID 0 に変更されるが、最初は -1 だった。

### 痕跡6：エディタが `sys save` でコアダンプする

```
   sys creat; tname       " create e.tmp for write
   spa
   sys save               " dump core on error!
```

`ed1.s`——世界最初の Unix エディタ `ed` の起源。一時ファイル `e.tmp` の作成に失敗したら、**即座にコアダンプ。** エラーハンドリングという概念が存在しない。

ファイルが作れなければ OS に制御を返す。Thompson にとって、「一時ファイルが作れない」は「回復不能」を意味した。

## Multics の子、Unix の父

PDP-7 Unix は **Multics の否定** から生まれた。

Multics（鑑定書 #034）は10,943ファイル、35年間稼働した「廃墟の神殿」だった。リング型プロテクション、動的リンク、仮想メモリ。正しく、美しく、そして巨大だった。

Thompson と Ritchie はその Multics を作っていた側の人間だ。AT&T が撤退を決めた時、彼らは理想の OS のために割り当てられた GE-645 メインフレームを失った。残されたのは、廊下の PDP-7 だけ。

**Multics が「全てを正しくやろう」としたのに対し、PDP-7 Unix は「動けばいい」で始まった。** 10プロセスの同時管理。3,550ファイルの上限。ファイル名は4ワード（8文字）まで。RB09 ディスク1台、約1メガバイト。

Peter Neumann は Unix を **「去勢された Multics」** と呼んだ。Thompson 自身も Multics からの影響を認めている。ディレクトリツリー、シェル、プロセスの概念——全て Multics にあった。Thompson はそれを **18KB のメモリに収まる形で再発明した。**

## 推定される経緯

**1969年3月**: AT&T が Multics プロジェクトから撤退。Thompson と Ritchie が GE-645 を失う。

**1969年前半**: Thompson が廊下の PDP-7 に Space Travel を移植。ファイルシステムのアイデアを温め始める。

**1969年夏**: 妻 Bonnie が息子を連れてカリフォルニアに帰省。3週間で OS・シェル・エディタ・アセンブラを書く。

**1970年**: Dennis Ritchie が合流。fork-exec モデルの提案。テキスト処理ツール `roff` の実装。

**1970〜71年**: 「Unix Book I」「Unix Book II」のリスティングが印刷される。

**1971年**: PDP-11 への移植。ここから Unix が本格的に拡散する。

**2011年10月12日**: Dennis Ritchie 死去（70歳）。Steve Jobs の死の6日後。

**2016年**: Norman Wilson のスキャンから Warren Toomey が復元プロジェクト開始。

**2019年10月**: 第二のノートブック発見、追加スキャン実施。

**2020年1月**: Living Computer Museum の PDP-7A 実機（S/N 113）で起動成功。

## 鑑定人所見

PDP-7 Unix は「種子」だ。

3,000行の PDP-7 アセンブリが、2026年の世界を動かす全ての Unix 系 OS の種子になった。Linux カーネルは3,600万行。macOS は数億行。Android は数千万行。それら全ての起源が、18KB のメモリしかない捨てられたマシンの上で、妻の帰省中に書かれた3,000行にある。

最も象徴的なのは、このコードが **紙から復元された** ことだ。Dennis Ritchie が2011年に亡くなった後、遺品の黒いバインダーに約190ページのリスティングが見つかった。それを OCR にかけ、コミュニティが手作業で修正し、2020年にシリアルナンバー113の PDP-7A 実機で起動した。**死者の遺品から、51年前のコードが蘇った。**

Multics（鑑定書 #034）は「廃墟の神殿」だった——崩れ落ちた夢の中に全 OS の設計図が眠る。ITS（鑑定書 #032）は MIT のハッカーたちの「遊び場」だった。Minix（鑑定書 #009）は教科書の中の「模型」だった。PDP-7 Unix はそれらの **前** にある。Multics の夢が砕けた場所から、Thompson が拾い上げた種子。それが世界中に根を張った。

シェルのプロンプト `ready` が全てを語っている。「準備できました」。57年前の、たった3,000行のコードが。
