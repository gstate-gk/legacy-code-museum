# 「一度解いたらもう面白くない」を解決した男たち——@マークの冒険者と、毎回違うダンジョン

## はじめに

GitHubの片隅に「始祖」のコードを見つけた。

**Rogue**。1980年、UC Santa Cruz の Michael Toy と Glenn Wichman が C 言語で書いたダンジョン探索ゲームだ。

36ファイル。約369KB。画面にはASCII文字だけが並ぶ。`@` がプレイヤー。`.` が床。`#` が通路。`D` がドラゴン。

グラフィックスはない。サウンドもない。しかしこのゲームには、それまでのゲームにはなかったものがあった——**毎回違うダンジョン**。

Colossal Cave Adventure（1976年）も Zork（1977年）も、一度解けば終わりだ。マップは固定されている。謎は変わらない。Toy は言った——「自分が楽しめるゲームを作りたかった。でも一度解いてしまえばもう面白くない」。

だからダンジョンを毎回生成することにした。乱数で部屋を配置し、通路を繋ぎ、モンスターとアイテムを散りばめる。二度と同じダンジョンは現れない。

この発想が **「ローグライク」** というジャンルを生んだ。Nethack、Diablo、Hades、Dead Cells——全ての「毎回違う冒険」の祖先が、この36ファイルの中にある。

## 「@」——今いる場所

なぜプレイヤーキャラクターは `@` なのか？

**"where you're at"（今いる場所）。**

ソースコードの定義——

```c
#define PLAYER      '@'
#define FLOOR       '.'
#define PASSAGE     '#'
#define DOOR        '+'
#define TRAP        '^'
#define STAIRS      '%'
#define GOLD        '*'
#define POTION      '!'
#define SCROLL      '?'
#define ARMOR       ']'
#define RING        '='
```

一文字で世界を表す。`!` がポーション、`?` がスクロール、`]` が鎧。この記号体系は45年後の今もローグライクの標準だ。

## 発掘された痕跡

### 痕跡1：3×3グリッドのダンジョン生成——プロシージャル生成の原型

ダンジョン生成アルゴリズムは驚くほどシンプルだ。

画面を **3×3の9グリッド** に分割する。各グリッドにランダムなサイズ・位置で部屋を配置する。0〜3部屋を「消去」してダンジョンに変化をつける。

```c
rnd(4)          /* 0〜3部屋を消去 */
rnd(10) < level-1  /* 深い階層ほど暗い部屋が増える */
rnd(15) == 0     /* 1/15の確率で迷路部屋 */
```

通路は9部屋の接続グラフからスパニングツリーを作り、余剰通路をランダムに追加する。直角に曲がる通路が、あのダンジョンの雰囲気を生み出す。

**全てのプロシージャル生成ゲーム——Minecraft のワールド、Diablo のダンジョン、No Man's Sky の惑星——の思想的起点がここにある。**

### 痕跡2：D&D のルールがそのままCに翻訳されている

モンスターのステータス定義——

```c
/* Name         CARRY FLAG    str, exp, lvl, amr, hpt, dmg */
{ "dragon",      100, ISMEAN, { XX, 5000, 10, -1, ___, "1x8/1x8/3x10" } },
{ "bat",           0, ISFLY,  { XX,    1,  1,  3, ___, "1x2" } },
```

`"1x8/1x8/3x10"` ——ドラゴンの攻撃は「8面ダイス1回 + 8面ダイス1回 + 10面ダイス3回」。D&D のダメージダイス表記がそのまま文字列として埋め込まれている。

セービングスロー——

```c
/* D&Dのセービングスロー: 20面ダイスを振って14以上なら成功 */
roll(1, 20) >= 14 + which - level/2
```

`roll(1, 20)` ——20面ダイスを1回振る。D&D のテーブルトップRPGのルールが、C言語の関数呼び出しに変換されている。

```c
int roll(int number, int sides) {
    int dtotal = 0;
    while (number--)
        dtotal += rnd(sides)+1;
    return dtotal;
}
```

### 痕跡3：ポーションの色は毎回ランダム——アイテム識別システム

Rogue の最も独創的な設計の一つ。

ゲーム開始時に27色のポーション、偽ラテン語のスクロール名、26種の宝石の指輪が **ランダムにシャッフル** される。

```c
char *rainbow[] = { "amber", "aquamarine", "black", "blue", "brown",
    "clear", "crimson", "cyan", "ecru", "gold", "green", ... "yellow" };
```

「青いポーション = 回復薬」という知識は、次のゲームでは無効になる。プレイヤーは毎回、アイテムを「鑑定」し直さなければならない。

スクロールの名前は音節をランダム連結して生成される——

```c
static char *sylls[] = {
    "a", "ab", "ag", "aks", "ala", "an", "app", "arg", "arze", "ash",
    "bek", "bie", "bit", "bjor", "blu", "bot", "bu", "byt", "comp", ...
};
```

2〜5単語、各1〜3音節。「ZELGO MER」「JUYED AWK YACC」のような偽ラテン語が生成される。

### 痕跡4：セーブファイルは即削除される——パーマデスの設計

Rogue は死んだらキャラクターが消える。セーブ＆ロードでやり直すことはできない。

実装は単純だ——**ロードした瞬間にセーブファイルを `unlink()` で削除する**。

Toy の言葉——

> *「痛くするためではなかった。すべての決断に重みを持たせるためだった。没入感を高めるために。」*

このパーマデスの設計が、ローグライクというジャンルの核心的定義の一つとなった。

### 痕跡5：when/otherwise/until——Cを別の言語に見せる遊び心

```c
#define when        break;case
#define otherwise   break;default
#define until(expr) while(!(expr))
#define hero        player.t_pos
```

`when`、`otherwise`、`until`——C言語のswitch文とwhile文を、別の言語のように見せるマクロ。`hero` は「プレイヤー座標」の別名。

1980年のプログラマーの遊び心が、マクロ定義に残っている。

### 痕跡6：墓石のアスキーアート——1980年が永遠に刻まれている

死亡すると表示される墓石——

```c
static char *rip[] = {
"                       __________\n",
"                      /          \\\n",
"                     /    REST    \\\n",
"                    /      IN      \\\n",
"                   /     PEACE      \\\n",
"                  /                  \\\n",
"                  |                  |\n",
"                  |   killed by a    |\n",
"                  |                  |\n",
"                  |       1980       |\n",
"         ________)/\\_//(\\/(/\\)/\\//\\/|_)_______\n",
    0
};
```

「killed by a ○○」。年号は **1980固定**——ゲームの誕生年が、全てのプレイヤーの墓石に刻まれる。

### 痕跡7：cursesの作者が文句を言いながら共同開発者になった

Ken Arnold は UC Berkeley で curses ライブラリを開発した人物だ。Vi（Bill Joy のエディタ）のコードを流用・改良し、端末を座標指定可能な画面として扱えるライブラリを作った。

Toy が BSD から curses を入手し、Rogue の開発に使った。Arnold は後に Rogue を発見した。

**自分のライブラリがひどい使われ方をしていることに苦言を呈しながらも、感銘を受け、開発チームに加わった。**

Arnold はソースコードの最適化と整理に貢献した。著作権表示に3人目の名前として刻まれている——

```c
/* Copyright (C) 1980-1983, 1985, 1999 Michael Toy, Ken Arnold and Glenn Wichman */
```

## Wichman はRogueをクリアできない

Glenn Wichman——Rogue の共同開発者、ゲームデザイナー、「Rogue」と命名した男——は、**自分のゲームをクリアしたことがない**。

妻はクリア済み。Toy は数回クリアしている。

Wichman の言葉——

> *「毎回違う冒険——これが何年もの間、あれほど人気だった本当の理由だ。」*

---

## Rog-O-Matic——Rogueを倒すAI

1981年、カーネギーメロン大学（CMU）の4人の学生が **Rog-O-Matic** を開発した。Rogue を自動プレイする AI だ。12,000行のC言語。端末ストリームをインターセプトしてコマンドを送信する。

1983年、2ヶ月間で106ゲームをプレイし、**全プレイヤー中最高のメジアンスコア** を達成した。

1984年の論文タイトル——「ROG-O-MATIC: A Belligerent Expert System（好戦的エキスパートシステム）」。

**Rogue は、最初の「AIベンチマークゲーム」の一つになった。**

## 推定される経緯

**1978年**: Toy と Wichman、UC Santa Cruz の新入生として出会う。

**1980年**: Rogue 完成。curses ライブラリで端末上に動的なダンジョンを表示。

**1981年**: 2BSD に収録。CMU で Rog-O-Matic 開発開始。

**1983年**: Epyx 社が商用版を発売（IBM PC版）。

**1984年**: 4.2BSD に同梱、世界の大学に拡散。

**1986年**: 4.3BSD でソースコード公開（チート防止のため長年非公開だった）。

**1987年**: NetHack 登場（Rogue → Hack → NetHack の系譜）。

**1996年**: Diablo 発売（David Brevik が Rogue を直接参照）。

**2008年**: Berlin Interpretation で「ローグライク」が定義される。

**2018年**: Hades、Dead Cells——ローグライトの時代。

## 鑑定人所見

Rogue は「ダイス」だ。

画面にはASCII文字しかない。`@` がプレイヤー。`.` が床。`D` がドラゴン。しかしこの文字の世界は、毎回違う。乱数が部屋を配置し、通路を繋ぎ、モンスターを散りばめる。二度と同じダンジョンは現れない。

最も象徴的なのは **アイテム識別システム** だ。27色のポーションが毎ゲームシャッフルされる。スクロールの名前は音節からランダム生成される。「知識」が通用しない世界——プレイヤーは毎回、世界を最初から理解し直さなければならない。この設計は、Diablo の「未鑑定アイテム」、Nethack の「ポーションの色テスト」として45年後の今も生き続けている。

D&D のルールが `roll(1, 20)` という関数呼び出しに変換されているのは、テーブルトップRPGからコンピュータRPGへの翻訳の最も初期の例だ。紙と鉛筆とダイスの世界が、C言語の `rnd()` と `roll()` に変わった。ゲームの本質は変わらない——不確実性と選択の連続。

セーブファイルの即削除は、「没入感のためのデザイン」という概念の最も純粋な実装だ。Toy は「痛くするためではない」と言った。全ての決断に重みを持たせるため。この設計思想は、45年後の「ソウルライク」「ローグライト」に受け継がれている。

そして Glenn Wichman——自分のゲームをクリアしたことがない男。毎回違うダンジョンを作った男が、自分のダンジョンを攻略できない。しかし彼の妻はクリアしている。

Multics（鑑定書 #034）は「廃墟の神殿」だった。Rogue は **「ダイス」** だ——振るたびに違う目が出る。そしてその目は、45年間振り続けられている。
