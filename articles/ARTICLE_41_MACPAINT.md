# 「今日マイナス2000行書いた」——存在しないものを発明し、コードを削って世界を変えた男

## はじめに

GitHubの片隅に「革命」のコードを見つけた。

**MacPaint**。1984年、Apple の Bill Atkinson が Pascal と68000アセンブリで書いた、世界初の本格的ビットマップペイントアプリケーションだ。

5,804行のPascal。2,738行のアセンブリ。合計約9,000行。

128KB の RAM に、OS と QuickDraw と MacPaint が同居する。浮動小数点演算はない。ハードディスクもない。400KB のフロッピー1枚。

この制約の中で、Atkinson はマウスで絵を描ける世界を作った。ブラシ、スプレー、テキスト、投げ縄、塗りつぶし——全てがモノクロの 512×342 ピクセルの画面に収まっている。

しかしこの男の最も有名な仕事は、**コードを書いたこと** ではない。

**「今日マイナス2000行書いた。」**

管理者が「今週何行書いた？」と聞いた。Atkinson は報告書に **-2000** と記入した。QuickDraw のリージョン計算を書き直し、6倍高速化し、2,000行削減したのだ。管理者はそれ以降、彼に行数を聞かなくなった。

## 存在しないものを見た男

1979年、Steve Jobs は Xerox PARC を訪問した。Bill Atkinson も同行していた。

Atkinson は PARC の Smalltalk のデモで **重複ウインドウ** を見た——ウインドウが重なり合い、前面のウインドウを動かすと背面のウインドウが再描画される。

帰り道、Atkinson はこれを実装しようと決意した。

しかし実は、**PARC は重複ウインドウの再描画を実装していなかった**。困難すぎる問題として放棄していたのだ。Atkinson は「存在しないもの」を見て、それを実装しようとした。

数ヶ月の代数的考察の末、Atkinson は **Regions（リージョン）** というアルゴリズムを発明した。任意の形状の領域を「反転ポイント」の配列として圧縮表現し、集合演算（和集合、差集合、XOR）を高速に実行する。これが LisaGraf——後の QuickDraw——の核心となった。

矩形のリージョンはわずか10バイト。複雑な形状でも効率的に格納できる。

```asm
RGNSIZE  .EQU    0   ;WORD, TOTAL BYTES
RGNBBOX  .EQU    2   ;RECT
RGNDATA  .EQU    10  ;START OF RGN DATA
```

## 発掘された痕跡

### 痕跡1：「リージョンの作り方は覚えてる」——死にかけた男の第一声

1982年春、Atkinson は朝の通勤中、疲労でコルベットを停車中のトラックに突っ込んだ。車の屋根がもぎ取られた。警官は「首を切断されてもおかしくなかった」と言った。

Jobs は事故を知り、即座に病院に駆けつけた。

> *Jobs: 「大丈夫か？ みんな心配したぞ。」*

苦痛の中で Atkinson は微笑んだ。

> *「心配しないで、Steve。リージョンの作り方はまだ覚えてるよ。」*
> *("Don't worry, Steve, I still remember regions.")*

**Regions のアルゴリズムは Atkinson の頭の中にしかなかった。** もし彼の記憶が失われていれば、Macintosh の核心技術が失われるところだった。

### 痕跡2：マイナス2000行の生産性

1982年初頭、Lisa 開発チームの管理者がプログラマーに毎週の「書いたコード行数」を報告させ始めた。

Atkinson は QuickDraw のリージョン計算ルーチンを完全に書き直した。よりシンプルで汎用的なアルゴリズムに置き換え、**6倍高速化**し、コードを **2,000行削減** した。

報告書に記入した数字——**-2000**。

Atkinson の見解——

> *「行数でソフトウェアの生産性を測ることは馬鹿げており、冗長で肥大化した壊れたコードを書くことを奨励するだけだ。」*

**管理者はそれ以降、彼に行数を聞かなくなった。**

### 痕跡3：128KBに全てを詰め込む——UnloadSeg の嵐

初代 Macintosh の RAM はわずか128KB。ここに OS、QuickDraw、そして MacPaint が同居しなければならない。

Atkinson の解決策——メインループで毎回、使っていない機能のコードセグメントをメモリから追い出す。

```pascal
REPEAT
    UnloadSeg(@InitOnce);       { 初期化    }
    UnloadSeg(@PrintDoc);       { 印刷      }
    UnloadSeg(@CutOrCopy);      { カット/コピー }
    UnloadSeg(@Paste);          { ペースト    }
    UnloadSeg(@Rotate);         { 回転      }
    UnloadSeg(@ShowPage);       { ページ表示  }
    UnloadSeg(@DrawPat);        { パターン   }
    UnloadSeg(@EditPat);        { パターン編集 }
    UnloadSeg(@ChooseBrush);    { ブラシ選択  }
    UnloadSeg(@Symmetry);       { 対称描画   }
    UnloadSeg(@ShowPicture);    { ヘルプ     }
    ...
    SystemTask;
    IF active THEN TrackCursor;
    IF GetNextEvent(everyEvent,theEvent) THEN ProcessTheEvent;
UNTIL quitFlag;
```

11個のセグメントを毎ループ追い出す。スタックは9KBに制限。

```pascal
heapTop := StackPtr - 9216;           { save 9K for stack }
SetApplLimit(Pointer(heapTop));
```

**128KBで世界初のペイントアプリを動かすとは、こういうことだ。**

### 痕跡4：MonkeyLives——猿のテスター

68000アセンブリの中に、奇妙な関数がある。

```asm
.FUNC Monkey
;  FUNCTION Monkey: BOOLEAN;
TST      MonkeyLives                ;IS THE MONKEY ACTIVE ?
SGE      4(SP)                      ;YES IF >= ZERO
NEG.B    4(SP)                      ;CONVERT TO PASCAL BOOLEAN
RTS
```

`MonkeyLives`（猿は生きているか？）。ランダムなイベントを生成してアプリに送り込む自動テスト——**Monkey Testing** だ。Mac チームはランダム入力で予期しないクラッシュを発見する「猿」を使っていた。

現代のモバイルアプリ開発で使われる Monkey Testing の概念は、1984年のこのコードにまで遡る。

### 痕跡5：「nasty ROM address」——自嘲のコメント

```asm
.PROC EjectReset
;  eject default drive and re-boot Macintosh
...
JMP      $40000A                    ;nasty ROM address
```

「nasty ROM address（汚い ROM アドレス）」。ハードコードされた ROM アドレスへの直接ジャンプでリブートする。Atkinson 自身が「汚い」と認めている——しかし128KBの世界では、こうするしかなかった。

### 痕跡6：方眼紙のアイコン——Susan Kare の $2.50

MacPaint のアイコンをデザインしたのは **Susan Kare** だ。

1983年1月、高校の友人 Andy Hertzfeld に誘われ、Apple II 1台と引き換えにアイコンデザインを始めた。パロアルトの画材店で **$2.50** の方眼紙ノートを購入し、32×32マスのグリッドに鉛筆で手描きした。

ペイントブラシ、ハサミ（カット）、指（ペースト）。起動時の笑顔 Mac。ゴミ箱。Chicago フォント。

**$2.50 の方眼紙から、コンピュータ史上最も影響力のあるアイコンが生まれた。** 原画は MoMA（ニューヨーク近代美術館）に所蔵されている。

### 痕跡7：PDP-10 への隠しオマージュ——ではなく、Jobs への直談判

MacPaint のソースコードが公開されるまでには、6年の闘いがあった。

2004年、Macintosh 20周年イベントで Don Knuth が「MacPaint のソースを見たい」と発言した。Andy Hertzfeld が Atkinson に連絡。しかし Apple 社内の承認が何度もブロックされた。

**2010年1月、Hertzfeld は Steve Jobs に直接会って相談した。翌日、Jobs は Apple の法務責任者に公開を指示した。**

2010年7月18日、Computer History Museum のウェブサイトで公開。サーバーがダウンするほどのアクセスが殺到した。

Atkinson のツイート——

> *「やった！ Apple が私のオリジナルの QuickDraw と MacPaint のソースコードを Computer History Museum に寄贈し、ダウンロードして読めるようにしてくれた。」*

## 6日でJobsを黙らせた男

Bill Atkinson は神経科学の博士課程に在籍していた。

Steve Jobs が払い戻し不可の航空券を一方的に送りつけ、3時間説得した。Atkinson が「Pascal が役に立つと思うなら証明してみろ」と言うと、Jobs は言った——「6日でやれ」。

Atkinson は6日で Apple II 用 Pascal を作った。

Jobs はそれ以来ずっと彼を尊敬した。

## 石の写真家

1990年、Atkinson は Apple を退社した。

HyperCard を作り、QuickDraw を作り、MacPaint を作り、マーチングアンツを発明し、メニューバーを考案した男は、自然写真家に転身した。

切断・研磨した石の接写写真を専門にした。2004年に写真集『Within the Stone』を出版した。

2025年6月5日、Bill Atkinson は膵臓がんで亡くなった。74歳だった。

Tim Cook の追悼——

> *「真のビジョナリーであり、その創造性と心と、Mac における画期的な仕事は、永遠に私たちにインスピレーションを与え続けるだろう。」*

## 鑑定人所見

MacPaint は「窓」だ。

文字しか表示できなかったコンピュータに、窓を開けた。マウスで絵が描ける。ブラシを選べる。パターンで塗りつぶせる。それまでのコンピュータが「計算機」だったのに対し、MacPaint は「キャンバス」だった。

最も象徴的なのは **Regions の物語** だ。Atkinson は Xerox PARC で「存在しないもの」を見た。PARC のチームが困難すぎると放棄した重複ウインドウの再描画を、見たと思い込んで、自ら発明した。「誤解」が革新を生んだ。

そして事故。車の屋根がもぎ取られ、首を切断されかけた男が、病院のベッドで最初に言った言葉が「リージョンの作り方は覚えてる」。このアルゴリズムは彼の頭の中にしかなかった。もし記憶が失われていれば、Macintosh は存在しなかったかもしれない。

-2000行の報告は、ソフトウェア開発の本質を突いている。コードを書くことは目的ではない。問題を解くことが目的だ。2,000行を削って6倍速くなるなら、それは2,000行書くより遥かに価値がある。

Susan Kare の $2.50 の方眼紙は、MoMA に所蔵されている。32×32 ピクセルのアイコンが近代美術として認められた。技術と芸術の境界が消えた瞬間だ。

Atkinson は Apple を去り、石の写真家になった。コンピュータの GUI を発明した男が、石の断面を撮影する。しかしよく考えれば、どちらも同じことをしている——**見えないものを、見えるようにする仕事**だ。

Zork I（鑑定書 #029）は「言葉の迷宮」だった。MacPaint は **「窓」** だ——文字しかなかった世界に開かれた、最初の窓。
