# 12万ドルの宇宙戦争——「やるやる詐欺」の男が世界最初のビデオゲームを作るまで

## はじめに

GitHubの片隅に「始まり」のコードを見つけた。

**Spacewar!**。1962年、MIT の PDP-1 上で Steve Russell らが PDP-1 アセンブリで書いた宇宙戦闘ゲームだ。2隻の宇宙船が中央の恒星の重力に引かれながら、魚雷を撃ち合う。

7,521行。18ビットワード。冷蔵庫3台分のコンピュータ。価格12万ドル——現在の約1億3000万円。

これが **世界最初のビデオゲーム** だ。

しかし物語はコードの中だけでは終わらない。開発者は「ナメクジ」と呼ばれる先延ばしの常習犯だった。仲間に言い訳を潰されてようやく手を動かした。そして完成したゲームは無償で配布され、著作権の主張すらなかった——やがてそれが、Atari を生み、ゲーム産業という巨大な産業を創ることになる。

---

## 冷蔵庫3台分のゲーム機

1961年9月、DEC PDP-1 が MIT Building 26 の2階「kludge room」に到着した。

| 項目 | 仕様 |
|:---|---:|
| ワードサイズ | 18ビット |
| メモリ | 4,096ワード |
| クロック速度 | 約187kHz |
| ディスプレイ | Type 30 CRT、1024×1024点 |
| 描画速度 | 20,000点/秒 |
| 価格 | $120,000（2025年換算 約$1,293,000） |
| 生産台数 | 53台 |

Steve Russell の回想——

> *「スイッチがあった。入れると気持ちのいいガチャンという音がする。1文字打つと小さなメッセージが返ってくる。ものすごいパワーを感じた。」*

この「ものすごいパワー」を何に使うか。MIT の Tech Model Railroad Club（TMRC）のハッカーたちは、すぐに答えを見つけた。

## 発掘された痕跡

### 痕跡1：「ナメクジ」の言い訳が尽きた日

Steve Russell のあだ名は **"Slug"（ナメクジ）**。先延ばしの常習犯だ。

1961年夏、Russell と Martin Graetz、Wayne Wiitanen の3人は、ケンブリッジの Hingham 通りのアパートで「コンピュータ玩具の理論」を議論していた。自称「Hingham Institute」。着想源は E.E. "Doc" Smith の SF 小説『レンズマン』と『スカイラーク』シリーズ——宇宙船が光線を撃ち合う冒険活劇だ。

Graetz の回想——

> *「キンボール・キニスンのせいだ。そしてリチャード・シートンのせいだ。」*
> （どちらもDoc Smithの小説の主人公）

PDP-1 が届けば、これを作れる。周囲は「Russell が書くべきだ」と圧力をかけた。しかし Russell は逃げ回った。

**最後の言い訳**——

> *「サイン・コサインルーチンがないし、書き方もわからないんだよね。」*

**Alan Kotok の決定打**——Kotok は DEC 本社（メイナード）まで車を走らせ、サイン・コサインルーチンのテープを入手した。MIT に戻ると Russell の前にドンと置いた。

> *「他に何か言い訳あるか？」*

Russell の回想——

> *「見回しても言い訳が見つからなかったので、腰を据えて計算するしかなかった。」*

1961年12月、ようやくコーディング開始。1962年2月、動く宇宙船が画面に現れた。

### 痕跡2：「高価なプラネタリウム」——天文学への執着

Russell が最初に実装した背景は、ランダムな点の星空だった。

**Peter Samson は怒った。**

Russell の証言——

> *「Peter Samson はランダムな星に怒った。」*

Samson は American Ephemeris and Nautical Almanac（米国天文暦）のデータを使い、ケンブリッジ上空の実際の夜空を再現した。赤緯22.5度N〜22.5度Sの帯に含まれる **5等星以上の全ての実在の星** を、相対的な明るさで表示する。

ソースコードの星図データ——

```
mark 1537, 371    /87 taur, aldebaran    （おうし座アルデバラン）
mark 1762, -189   /19 orio, rigel        （オリオン座リゲル）
mark 1990, 168    /58 orio, betelgeuze   （オリオン座ベテルギウス）
mark 2280, -377   /9 cmaj, sirius        （おおいぬ座シリウス）
mark 2583, 125    /25 cmin, procyon      （こいぬ座プロキオン）
mark 4842, 448    /16 boot, arcturus     （うしかい座アークトゥルス）
mark 6747, 196    /53 aqil, altair       （わし座アルタイル）
```

星図はゆっくりスクロールする。12万ドルのコンピュータで実際の夜空を再現する——名付けて **"Expensive Planetarium"（高価なプラネタリウム）**。

ランダムな星では許せない。天文学カタログに忠実でなければならない。この執着こそ、ハッカー文化の本質だ。

### 痕跡3：重力——計算時間を「捻り出した」男

Dan Edwards は、ゲームに重力を入れたかった。画面中央の恒星が宇宙船を引き寄せる。しかし問題があった——PDP-1 の処理速度では、重力計算を追加する余裕がなかった。

Edwards の解決策は2段階だった。

**第1段階：アウトラインコンパイラ**——宇宙船の形状データをフレームごとに解釈する代わりに、描画用のマシンコードに動的にコンパイルして実行する。**1962年の JIT コンパイル**だ。

**第2段階：空いた計算時間で重力を実装**——

```asm
lac .          / calc if collision
sub .          / delta x
spa            / take abs value
cma
dac ~mt1
```

18ビットワードの制約の中で、逆二乗則を近似する。距離の計算にはシフト演算を多用し、乗算命令がない PDP-1 では17回の `mus`（multiply step）を繰り返すソフトウェア乗算で二乗を計算した。

この改良により、ゲームは「面白い暇つぶし」から「深い戦術的ゲーム」に変貌した。安定軌道、スリングショット機動、重力を利用した回避——戦略の幅が爆発的に広がった。

### 痕跡4：ハイパースペース——パニックボタンの確率論

Martin Graetz が設計した「最終手段」。追い詰められた時に発動すると宇宙船が消え、数秒後にランダムな位置に再出現する。

しかしタダでは使えない。

- **使うたびに爆発確率が上昇**——1回目は1/8、2回目は2/8、3回目は3/8……
- 太陽の中に再出現する可能性もある
- **「8回目のボタンを押す者はいない」**

ソースコードのパラメータ——

```
law i 10    / number of hyperspace shots    （ハイパースペース使用回数）
law i 40    / time in hyperspace before breakout （滞在時間）
40000       / hyperspatial uncertancy       （不確定性）
```

注目すべきは最後の行——**"uncertancy"**。"uncertainty" のスペルミスだ。1962年のプログラマーもtypoをする。そしてそのtypoは、60年以上経った今もソースコードに残っている。

### 痕跡5：静音コントローラー——世界最初のeスポーツ的発想

Alan Kotok と Bob Saunders は、TMRC のパーツ箱から木製のコントロールボックスを製作した。

PDP-1 のトグルスイッチでは肘が疲れる。だが彼らがコントローラーを作った真の理由は別にあった——ベークライトの天板に **静音スイッチ** を装着したのだ。

**相手に操作音を聞かれないため。**

1962年にして、対戦ゲームにおける「情報の秘匿」という概念が生まれていた。これは **世界最初のゲーム用コントローラー** であり、同時に **eスポーツ的思考の萌芽** でもある。

### 痕跡6：5命令の乱数生成器——全てのランダムを5行で

Spacewar! の全てのランダム要素——ハイパースペースの再出現位置、爆発の破片、星のブリンク——は、たった5命令の乱数生成器から生まれる。

```asm
lac ran
rar 1s
xor (355760
add (355670
dac ran
```

線形フィードバックシフトレジスタ（LFSR）。ロード、シフト、XOR、加算、ストア。18ビットの世界で、これ以上削りようがない。

### 痕跡7：無償配布——著作権という概念すらなかった時代

1962年春、開発者たちは DEC のカンファレンスで Spacewar! のソースコードを提供した。

ロイヤリティの請求は検討すらされなかった。理由は2つ——

1. **ハッカー倫理に反する**: 情報は自由であるべきだ
2. **法的に未知の領域**: 12万ドルの業務用コンピュータのプログラムに著作権が適用されるか、誰にもわからなかった

DEC は Spacewar! を **工場テストに使い始めた**。CRT、演算装置、メモリを全て酷使するこのゲームは、優れた総合診断プログラムでもあった。やがて **全 PDP-1 に出荷時プリロード** されるようになった。

**世界最初のバンドルソフトウェア。** そしてそれは無償だった。

## Spacewar! から Atari へ

Spacewar! の影響は MIT の外へ広がった。

1960年代後半、スタンフォードの AI 研究所で一人の学生が PDP-6 上の Spacewar をプレイしていた。**Nolan Bushnell**。

1971年、Bushnell は Spacewar のアーケード版「**Computer Space**」を開発した。しかし操作が複雑すぎて商業的に失敗した。

1972年、Bushnell は **Atari** を設立。今度はシンプルなテニスゲーム「**Pong**」を作った。これが世界初の商業的に成功したビデオゲームとなった。

Spacewar! → Computer Space → Pong → ゲーム産業。

12万ドルのコンピュータで、ナメクジと呼ばれた男が無償で作ったゲームが、数兆ドルの産業を生んだ。

**1971年**: Nolan Bushnell が Computer Space を開発（商業的失敗）。

**1972年**: Bushnell が Atari を設立、Pong をリリース。

**2005年**: Computer History Museum が PDP-1 を復元し、Spacewar! を実動デモ。

## 鑑定人所見

Spacewar! は「始まり」だ。

12万ドルのコンピュータの前で、ナメクジと呼ばれた男が言い訳を並べていた。サイン・コサインルーチンがない。書き方がわからない。仲間がDEC本社からテープを持ってきて、「他に言い訳あるか？」と突きつけた。

そこから生まれた7,521行のコードは、驚くべき密度で「最初」を詰め込んでいる。世界最初のビデオゲーム。最初のゲーム用コントローラー。最初のバンドルソフトウェア。そして恐らく、最初のオープンソースソフトウェアの一つ。

最も印象的なのは、**Dan Edwards のアウトラインコンパイラ**だ。重力を実装するために計算時間が足りない。普通なら重力を諦める。Edwards は逆に、描画ルーチンを動的にコンパイルして高速化し、空いた時間で重力を計算した。1962年の JIT コンパイル——Java の32年前に。

そしてもう一つ、**Peter Samson の天文学的執着**。ランダムな星では許せない。American Ephemeris のデータを手で入力し、実在の星をその明るさで表示する。「高価なプラネタリウム」。この執着は無意味だったか？ ゲームの本質には関係ない。しかしこの「正しくあろうとする衝動」こそが、ハッカー文化の核心だ。

Spacewar! の開発者たちは著作権を主張しなかった。ハッカー倫理に反するから。そしてプログラムに著作権が適用されるかどうか、誰にもわからなかったから。この無償のコードが、Nolan Bushnell を経て Atari を生み、Pong を生み、数兆ドルのゲーム産業を生んだ。

ソースコードには60年前のtypoが残っている——"hyperspatial uncertancy"。uncertainty のスペルミス。MIT のハッカーたちも人間だった。そしてそのtypoは、コードが一度も「修正」されていない証拠でもある。1962年のコードが、そのまま2026年のGitHubに眠っている。

ELIZA（鑑定書 #019）は「鏡」だった。SHRDLU（鑑定書 #026）は「箱庭」だった。Spacewar! は **「遊び場」** だ——12万ドルの遊び場。そしてその遊び場から、全てが始まった。
