# 100ドルのソフトが2,000ドルのコンピュータを売った——6502アセンブリで「表計算」を発明した二人

## はじめに

GitHubの片隅に「発明」のコードを見つけた。

**VisiCalc**。1979年、Dan Bricklin と Bob Frankston が6502アセンブリで書いた世界初のスプレッドシートだ。

約3,600行（逆アセンブル復元版）。アセンブリ 95%。**32KB のメモリに、表計算という概念そのものを詰め込んだ。**

始まりはハーバード・ビジネス・スクールの授業だった。教授が黒板に縦横の線を引き、セルに数字を書き込んでいた。パラメータを変更するたびに、消しゴムで消して書き直す。Bricklin はその光景を見て思った——**「これ、コンピュータでやれるのでは」。**

Bricklin は MIT 時代の友人 Bob Frankston に声をかけた。二人は Software Arts を設立し、1978年の冬から1979年初頭にかけて、**2ヶ月で** VisiCalc を書き上げた。

Frankston が6502アセンブリでコードを書いた。**アセンブラは Multics 上で動いていた。** PDP-7 Unix（鑑定書 #043）の祖先であり、Multics（鑑定書 #034）の直系のタイムシェアリングシステムの上で、Apple II のプログラムが生まれた。

1979年10月17日リリース。価格は100ドル未満。Apple II は約2,000ドル。しかし **1979年に売れた Apple II の25%以上が、VisiCalc を使うために購入された。** 100ドルのソフトウェアが、2,000ドルのハードウェアを売った。世界初の「キラーアプリ」だ。

## 発掘された痕跡

### 痕跡1：逆アセンブルから蘇ったコード

```assembly
;---------------------------------------------------------*
; Disassembled with The Flaming Bird Disassembler          *
; (c) Phoenix corp. 1992,93 - All rights reserved          *
;---------------------------------------------------------*
```

このソースコードはオリジナルではない。Brutal Deluxe Software が The Flaming Bird Disassembler を使って、VisiCalc のバイナリから **逆アセンブル** したものだ。

逆アセンブラが解析できなかった2つのアドレス範囲（$2356–$2E97 と $49D3–$57B3）は、da65 で再逆アセンブリし、**62個の進入点** を含む重要なルーチンが復元された。さらに約600行が不正な命令として誤認識されていたのを、修正スクリプトで修正。**文字列テーブル約45個、ルックアップテーブル3種、関数名テーブル（SUM、SIN、COS 等）が復元された。**

全ての修正は「バイト同一のバイナリを生成する」ことを保証している。**一文字たりとも創作は許されない。コード考古学の原則だ。**

### 痕跡2：32KB に世界を詰め込む

```assembly
KBD       = $C000
KBDSTROBE = $C010
SPKR      = $C030
HOME      = $FC58
COUT      = $FDED
```

Apple II のハードウェアアドレス直接参照。キーボード、スピーカー、画面出力——全てがメモリマップド I/O。OS という概念がほぼ存在しない世界。

プログラム本体は $0800〜$6FFF に展開される。約27KB。残りの5KB がユーザーのデータ領域。**256行 × 64列のグリッドを、5KB のメモリで管理する。** セルの大半は空だから成立する——疎行列の考え方を、1979年に実装していた。

### 痕跡3：キーボードを取りこぼさない

Frankston は Apple II のキーボードと格闘した。ユーザーが速くタイプすると、**文字が消える。** Apple II のキーボードバッファは1文字分しかなかった。

解決策：VisiCalc は **常にキーボードを監視し続けた。** 他の処理中もキーボードをポーリングし、入力を取りこぼさないようにした。CPU リソースを消費するが、ビジネスユーザーにとって「入力した文字が消える」は許容できなかった。

**ハードウェアの制約を、ソフトウェアの力技で補う。** Photoshop 1.0（鑑定書 #042）が仮想メモリを自前実装し、DOOM（鑑定書 #044）がゾーンメモリを自前管理したのと同じ精神だ。

### 痕跡4：Multics の上で Apple II のコードを書く

VisiCalc の開発ツールチェーンは驚くべき構成だった。

Bricklin と Frankston は銀行からローンを組んで **Prime ミニコンピュータ** を購入した。その上で **Multics タイムシェアリングシステム** を動かし、エディタ、アセンブラ、リンカを自作した。そこで6502アセンブリを書き、Apple II 向けにクロスアセンブルした。

**Multics（鑑定書 #034）の上で、Apple II のキラーアプリが生まれた。** 1960年代の巨大OS の子孫が、1970年代のパーソナルコンピュータ革命を生む道具になった。Unix は Multics の「去勢版」だった。VisiCalc は Multics を「踏み台」にした。

### 痕跡5：特許が取れなかった発明

Bricklin と Frankston は特許弁護士に相談した。答えは——「ソフトウェアは特許の対象にならない」。

弁護士は成功率を **10%** と見積もった。「実際にはソフトウェアであることを隠す」テクニックを使っても。コストを考慮し、二人は特許出願を断念した。

**「表計算」という概念を発明した人間が、その発明を法的に保護できなかった。** Lotus 1-2-3 が1983年に登場し、IBM PC の256KB メモリと80カラム画面を活用して VisiCalc を圧倒した。VisiCalc の月間販売数は39,000本から5,700本に急落。1985年、Lotus Development が Software Arts を買収し、VisiCalc の販売を終了した。

発明者が市場から追い出される。**特許を持っていれば、歴史は違っていた。**

### 痕跡6：ビープ音とクリック音——UI の原型

```assembly
; Sound bell
L0854: LDA #$40         ; Delay value
       JSR L0875         ; Call delay routine
       LDY #$C0          ; Loop counter
L085B: LDA #$0C          ; Delay value
       JSR L0875         ; Call delay routine
       LDA SPKR          ; Toggle speaker
       DEY                ; Decrement counter
       BNE L085B          ; Continue loop
       RTS
; Sound click
L0867: LDY #$08          ; Loop counter
L0869: LDA #$28          ; Delay value
       JSR L0875         ; Call delay routine
       LDA SPKR          ; Toggle speaker
       DEY                ; Decrement counter
       BNE L0869          ; Continue loop
       RTS
```

スピーカーを直接トグルしてビープ音とクリック音を生成する。`$C030`（スピーカー）のメモリアドレスを読むだけで、スピーカーの膜が動く。

**エラー時にビープ、入力受付時にクリック。** ユーザーにフィードバックを返す。GUI が存在しない時代の「ユーザー体験設計」が、6502アセンブリ20行に詰まっている。

## 黒板の授業が変えた世界

VisiCalc は、一人の MBA 学生が黒板の授業を見て思いついた。

しかし「思いつく」ことと「作る」ことの間には深い溝がある。Bricklin のアイデアを形にしたのは Frankston の6502アセンブリ力だった。**32KB のメモリで、再計算が動的に走り、カーソルが即座に移動し、数字が目の前で変わる。** 紙の上では不可能だった「What-if 分析」が、初めてリアルタイムで可能になった。

Steve Wozniak は語っている——**「小規模ビジネスが Apple II の90%を購入した」。** VisiCalc がなければ、Apple II は趣味のマシンのままだった可能性がある。パーソナルコンピュータが「仕事の道具」になった瞬間、そこには VisiCalc があった。

**1985年**: Lotus Development が Software Arts を買収、VisiCalc 販売終了。

**累計販売**: 約100万本（6年間で70万本超）。

## 鑑定人所見

VisiCalc は「原器」だ。

メートル原器がメートルを定義したように、VisiCalc は **「スプレッドシート」** を定義した。行と列のグリッド。セル参照。自動再計算。What-if 分析。これらの概念は VisiCalc が発明し、47年後の2026年も変わっていない。Excel も Google Sheets も、VisiCalc が定義したパラダイムの中で動いている。

最も象徴的なのは **「特許が取れなかった」** ことだ。1979年、ソフトウェアは特許の対象にならなかった。「表計算」という概念を発明した人間が、その発明を法的に保護できず、後発の Lotus 1-2-3 に市場を奪われた。発明者が報われない。**しかし発明は残った。**

Elite（鑑定書 #041）は「星図」だった——6バイトから宇宙を展開する。VisiCalc は **「原器」** だ——3,600行の6502アセンブリが、「表計算」という概念の原器になった。メートルが変わらないように、スプレッドシートの基本設計は47年間変わっていない。
