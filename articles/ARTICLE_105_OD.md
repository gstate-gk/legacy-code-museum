# `base = 010`——自分自身も8進数で初期化するodと、PDP-11の語単位で世界を読む1979年のバイナリダンパー

## はじめに

```c
int base = 010;
```

`base` の初期値が `010`——C言語の整数リテラルで `010` は8進数の8だ。**8進数ダンプのプログラムが、自分の動作モードの初期値を8進数で書いている**。意図的かどうかは分からないが、1979年のBell Labsらしい自然さだ。

`od` は "octal dump" の略。バイナリファイルの中身を人間が読める形に変換するデバッグツールだ。オプションなしで実行すれば8進数で出力する——だから `od` という名前で、だから `base = 010` なのだ。

Bell-32V（1979年）の `od.c` は291行。読む・変換する・出力するという最小限の機能を、5つのビットフラグと1つの再帰関数で実装した。

## `unsigned short word[8]`——PDP-11の語単位

```c
unsigned short  word[8];
unsigned short  lastword[8];
```

odはファイルを`unsigned short`（16bit整数）の配列として読む。

```c
n = fread((char *)word, 1, sizeof(word), stdin)
```

`sizeof(word) = 16`バイト——1回の読み込みで8ワード（16バイト）。これはPDP-11アーキテクチャの反映だ。PDP-11は16bitワードマシンで、メモリへのアクセスも語（word）単位が自然だった。バイト単位ではなく語単位でバイナリを扱う——1979年のUnixはまだPDP-11の世界だった。

## `conv` ビットマスク——5つのフォーマットが同時に走る

```c
int conv;

case 'o': conv |= 001; f = 6; break;  /* 8進数（6桁幅） */
case 'd': conv |= 002; f = 5; break;  /* 10進数（5桁幅） */
case 'x':
case 'h': conv |= 010; f = 4; break;  /* 16進数（4桁幅） */
case 'c': conv |= 020; f = 7; break;  /* 文字表現（7桁幅） */
case 'b': conv |= 040; f = 7; break;  /* バイト8進数（7桁幅） */
```

各フォーマットは `conv` のビット1つに対応する。`od -ocx` と実行すれば `conv = 001 | 001 | 010 = 011`——8進数と16進数を同時出力する。

`case 'x': case 'h':` ——hexフォーマットに `x` と `h` の2つの名前がある。どちらを使っても同じビットが立つ。歴史的にhexは `h` で書かれることもあった。

`f` は各フォーマットの出力幅（桁数）。`max` に最大幅を記録し、`pre()` で列幅を揃える。複数フォーマットが同時出力されるとき、アドレスは1回だけ、各フォーマットはタブで区切って並べる——`line()` 関数がビットシフトで `conv` を走査する。

```c
for(c=1; c; c<<=1) {       /* c = 1, 2, 4, 8, 16, 32 ... */
    if((c&conv) == 0)
        continue;           /* このビットが立っていなければスキップ */
    ...
    putx(w[i], c);          /* 対応フォーマットで出力 */
}
```

`c<<=1` で1ビットずつシフトしながら全フォーマットを試す——ビットマスクのイテレーション。

## `*`——同一ブロックの圧縮

```c
same = -1;
for ( ; (n = fread(...)) > 0; addr += n) {
    if (same >= 0) {
        for (f=0; f<8; f++)
            if (lastword[f] != word[f])
                goto notsame;
        if (same == 0) {
            printf("*\n");  /* 最初の重複で * を1行だけ出力 */
            same = 1;
        }
        continue;           /* 同一ブロックは読み飛ばす */
    }
notsame:
    line(addr, word, ...);
    same = 0;
    for (f=0; f<8; f++)
        lastword[f] = word[f];  /* 前のブロックを記憶 */
}
```

前の16バイトブロックと完全一致すれば `*` を1行だけ出力してスキップする。ゼロで埋められた大きなファイルを `od` しても、`*` の1行で表現される——現代の `od` でも同じ動作だ。

`same = -1` が「まだ前ブロックなし」、`same = 0` が「前ブロックあり（まだ `*` を出していない）」、`same = 1` が「`*` を出力済み」の3状態。`goto notsame` で差異が見つかった瞬間に通常出力に切り替える。

## `putn()` ——dc と同じ再帰的桁出力

```c
putn(n, b, c)
long n;
{
    register d;

    if(!c)
        return;
    putn(n/b, b, c-1);     /* 上位桁を先に再帰 */
    d = n%b;
    if (d > 9)
        putchar(d-10+'a');  /* 16進数: a〜f */
    else
        putchar(d+'0');     /* 0〜9 */
}
```

引数 `c` は桁数。`putn(255, 8, 3)` で `377` を出力する——再帰で上位桁から順に展開する。これは dc（#068）の `oprint()` と全く同じパターンだ。dcもodも、Bell Labsで共有されていた「再帰で基数変換する」イディオムを使っている。

`d > 9` で `'a'〜'f'` に変換——16進数にも対応する汎用実装。`putn(addr, base, 7)` でアドレスを出力し、`putn((long)n, 8, 6)` でデータを8進数6桁で出力する。

## `cput()` ——`-c` オプションの文字変換テーブル

```c
cput(c)
{
    c &= 0377;              /* 8bit マスク */
    if(c>037 && c<0177) {   /* 表示可能文字 */
        printf("  ");
        putchar(c);
        return;
    }
    switch(c) {
    case '\0': printf(" \\0"); break;
    case '\b': printf(" \\b"); break;
    case '\f': printf(" \\f"); break;
    case '\n': printf(" \\n"); break;
    case '\r': printf(" \\r"); break;
    case '\t': printf(" \\t"); break;
    default:   putn((long)c, 8, 3); /* その他は8進数3桁 */
    }
}
```

`0x20`〜`0x7e`（表示可能ASCII）はそのまま出力。制御文字は `\0`・`\b`・`\f`・`\n`・`\r`・`\t` の6種類のエスケープシーケンスで表現し、それ以外は8進数3桁で出力する。

`-c` フラグはワードを2バイトに分解して `cput()` を2回呼ぶ：

```c
case 020:   /* -c フラグ */
    {
        unsigned short sn = n;
        cput(*(char *)&sn);           /* 下位バイト */
        putchar(' ');
        cput(*((char *)&sn + 1));     /* 上位バイト */
        break;
    }
```

`unsigned short` に格納したワードをポインタキャストで2バイトに分解する——1979年のtype punning。バイト順はマシン依存だ（PDP-11はリトルエンディアン）。

## `offset()` ——柔軟な開始位置指定

```c
offset(s)
register char *s;
{
    if (*s=='+') s++;
    if (*s=='x') { s++; base = 16; }
    else if (*s=='0' && s[1]=='x') { s += 2; base = 16; }
    else if (*s == '0') base = 8;
    ...
    while(*p) {
        if (*p++=='.') base = 10;  /* '.' があれば10進数 */
    }
    ...
    if(*s=='b' || *s=='B') a *= 512;  /* 'b' でブロック単位 */
    fseek(stdin, a, 0);
    addr = a;
}
```

`+100`（10進数）、`+0100`（8進数）、`+0x100`（16進数）、`+100.`（10進数を `.` で明示）、`+2b`（2×512=1024バイト）——開始オフセットの書き方を5通り解釈する。`fseek(stdin, a, 0)` でファイルの任意位置から読み始める。

## デフォルトが8進数だった理由

1979年、コンピュータ科学者がバイナリを読む主要な方法は8進数だった。PDP-11の機械語命令は3ビット区切りで意味を持ち、8進数で書けば命令フィールドが自然に対応する。`ls -l` のパーミッション表示（`755`、`644`）も8進数だ。

16進数が優勢になるのはx86の普及（1980年代中盤）以降。それまでの「バイナリを見る道具」は自然と8進数を基本とした——だから `base = 010` なのだ。

**odは「8進数の時代の化石」だ。**

`base = 010` という行に、1979年のコンピュータ科学者の世界観が凝縮されている。PDP-11の語単位、8進数のパーミッション、3ビット区切りの機械語——全てが8進数を自然な表現として使っていた時代。x86とhexdumpの時代が来ても、`od -o` の出力と `*` による繰り返し圧縮は今日も変わらない。291行が50年後も動き続けている。
